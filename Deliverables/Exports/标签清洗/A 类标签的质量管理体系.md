#### 模块2：抽样层设计（解决“随机度低、关键样本覆盖不全”，符合统计学）
核心目标：基于“2W座场站（8省区+头部属性）”，采用**分层随机抽样**（比简单随机抽样更科学，确保关键样本覆盖），同时明确二次抽检样本池规则。

### 2.1 首次抽样：分层随机抽样（核心步骤+统计学依据）
#### 2.1 首次抽样：分层随机抽样（核心步骤+统计学依据）
1.  **步骤1：划分抽样层（解决“关键样本覆盖不全”）**  
    按“省区（8个）+ 是否头部站（2类）”交叉分层，共形成 `8×2=16个抽样层`（比如“华东省区-头部站”“华北省区-非头部站”）。  
    *   理由：头部站通常是业务核心，若用简单随机抽样可能漏抽；按省区分层可避免某一省区样本过少，确保区域代表性。

2.  **步骤2：计算总样本量（统计学公式，确保代表性）**  
    已知总体N=20000座，按行业通用标准（置信度95%、误差范围±5%），用以下公式计算总样本量n：  
    ```
    n = (Z² × P × (1-P)) / E²  
    （Z=1.96（置信度95%对应值）；P=0.5（最坏情况，样本比例未知时取0.5）；E=0.05（误差范围））
    ```  
    计算结果：n≈384座/周。  
    *   灵活调整：若头部站总数少（比如仅2000座），可单独提高头部站样本占比（比如总样本384座中，头部站占20%=77座，非头部站307座），确保核心样本覆盖。

3.  **步骤3：分配各层样本量（按“分层比例抽样法”）**  
    先统计每个层的“层总体数量”（比如“华东省区-头部站”共1500座，占总头部站2000座的75%），再按比例分配样本量：  
    *   示例：头部站总样本77座，“华东省区-头部站”分配 77×(1500/2000)=58座；“华北省区-头部站”分配77×(500/2000)=19座。  
    *   优势：每层样本量与层总体规模匹配，避免小层被忽略。

4.  **步骤4：层内随机抽样（确保随机度）**  
    对每个层内的场站按“场站ID”排序，用**随机数生成器**抽取样本：  
    *   工具：Excel的`RAND()`函数（生成0-1随机数，按随机数排序取前N个）；或在线随机数工具（如https://www.random.org/）。  
    *   避免人为干预：抽样过程全程留痕（记录随机数种子、排序结果），存储到【抽检任务表】的“抽样日志”字段。

##### 2.2 二次抽检：样本池设定（结合“质量监控+标签稳定性”目标）
假设二次抽检核心目标是“复核首次结果准确性+监控头部站标签稳定性”，样本池按以下规则构建：
| 二次抽检类型       | 样本池构成                                                                 | 抽样比例/数量 | 触发周期       |
|--------------------|----------------------------------------------------------------------------|---------------|----------------|
| 质量复核样本池     | 首次抽检结果中，随机抽取10%（含重叠站和独家站，确保复核无偏差）             | 38座/周（384×10%） | 与首次抽检同步 |
| 头部站跟踪样本池   | 首次抽检未抽到的头部站+首次抽到但判定为“重叠站”的头部站                     | 50座/周       | 每4周1次（月度） |
| 异常结果样本池     | 首次抽检中“核查存疑”（如数据缺失导致无法判定）的场站                       | 100%抽取      | 首次抽检完成后24小时内 |

*   理由：头部站是核心资产，需高频跟踪；异常结果必须100%复核，避免标签错误。


#### 模块3：过程管控层设计（实现“质量+进度双控”）
核心通过“**节点校验+异常预警+质量度量**”实现管控，适配零基础的手动操作场景。

##### 3.1 进度可控：节点预警与追踪
1.  **建立“进度看板”**（用Excel/飞书表格制作，实时更新）：  
    | 字段         | 说明                                                                 |
    |--------------|----------------------------------------------------------------------|
    | 抽检周期     | 2024W45（2024年第45周）                                              |
    | 计划样本量   | 384座                                                                |
    | 已抽样数量   | 384座（100%）                                                        |
    | 已核查数量   | 300座（78%）                                                        |
    | 逾期未核查   | 84座（列出场站ID、责任人、逾期时长）                                 |
    | 复核完成率   | 20座/38座（53%）                                                     |
    | 标签同步状态 | 未开始                                                               |
2.  **逾期预警规则**：  
    - 核查节点：逾期超过24小时，由质量负责人向核查员发送提醒（飞书/企业微信消息）；  
    - 全流程：周五18:00前未完成标签同步，触发“周度抽检异常报告”（同步给负责人）。

##### 3.2 质量可控：从“核查→复核→改进”闭环
1.  **核查质量管控（前端预防）**：  
    - 统一“核查操作手册”：明确3个核心步骤（附示例）：  
      ① 数据源获取：从“供给站数据表”中，按“场站地址→覆盖半径5公里”筛选供给站；  
      ② 数据清洗逻辑：剔除“供给站状态为已下线”“覆盖半径模糊”的数据；  
      ③ 判定规则：清洗后仍有≥1个供给站→重叠站，否则→独家站。  
    - 核查员培训：每周抽样前1小时，培训“操作手册”更新点（比如数据源变更）。

2.  **复核质量管控（中端校验）**：  
    - 复核人独立核查：不看首次核查结果，重新按“操作手册”执行核查，对比两次结果；  
    - 计算“核查准确率”：准确率=（两次结果一致的样本数/复核样本数）×100%；  
    - 准确率阈值：≥98%为合格，＜98%需：① 分析错误原因（如数据源错、清洗逻辑理解偏差）；② 对该批次所有样本二次复核。

3.  **质量改进（后端优化）**：  
    - 周度质量报告：记录“准确率、错误原因TOP3、改进措施”（比如“数据源缺失导致5个样本错判，下周协调业务方补充数据”）；  
    - 操作手册迭代：每月更新1次，将常见错误点加入“注意事项”（比如“供给站覆盖半径需以业务系统最新数据为准”）。


#### 模块4：数据回收与追溯层设计（实现“全链路可trace”）
核心是“**数据全生命周期记录+结构化存储**”，确保1年内可追溯。

##### 4.1 追溯链路设计（覆盖“数据→结果→变更”）
| 追溯维度       | 记录内容（存储在【抽检结果表】）                                                                 | 追溯方式（示例）                                                                 |
|----------------|--------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------|
| 数据来源       | 供给站数据来源：“团游供应商系统V2.0-接口名称：supply_station_list-调用时间：2024-11-01 09:00”      | 输入“场站ID+抽检周期”，可查到该样本用的数据源接口日志                            |
| 清洗逻辑       | “剔除状态=已下线的供给站（筛选条件：status!='off'）；填充覆盖半径为空的记录为5公里”                | 输入“核查ID”，可查到清洗用的SQL条件/Excel筛选步骤                                |
| 核查结果       | 判定结果：重叠站；依据：筛选出3个有效供给站（供给站ID：S1001/S1002/S1003）                        | 输入“场站ID”，可查到判定时的供给站明细                                          |

##### 4.2 数据存储与回收规则
1.  **存储介质**：初期用Excel（按“抽检周期”建文件夹，每周1个压缩包）；中期迁移到开源数据库（设置“存储有效期=1年”，自动清理超期数据）。  
2.  **回收权限**：仅开放“数据运营、质量负责人、数据开发”3类角色查看权限，确保数据安全。  
3.  **追溯工具**：初期用Excel的“筛选+查找”功能；中期在低代码平台搭建“追溯查询页面”（输入场站ID/抽检周期即可查全链路信息）。
