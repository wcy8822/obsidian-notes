### 1）一句话结论

明天的商户拜访，重点是“用一套标准化的【访谈提纲 + 产品说明 + 简易收益测算】去验证站点意愿和履约能力”，你需要带齐资料、想清问题、规划好会中分工和会后动作。

---

### 2）三段论据（现状与问题 → 关键洞察 → 路径方案）

**（一）先想清楚你要从商户这里“打捞什么信息”**

- 从BRD/主线看，非油线上化的空间、目标用户、商户分层都已经有清晰假设：全国非油渗透4–5%，目标对标8–10%；重点是有便利店/有履约能力的KA/CKA站，线上化弥补他们现在“有货但没渠道”的缺口。
- 你这次线下拜访，本质是验证三块：① 商户现状和痛点（非油现在怎么卖、哪里卡）；② 随手购这套产品对他们到底有没有吸引力；③ 他们有没有真实的履约能力和资源（人手/动线/库存/系统/合规）去跑“10分钟交付”的SOP。
- 所以不能只带“招商话术”，更重要的是带着明确的“调研问题清单”，现场把信息补齐，为后面选站、定政策、做收益模型和SLA判断提供一手事实。

**（二）基于现有PRD/BRD，把“随手购是什么、能给你带来什么钱”讲清楚**

- PRD已经把随手购的产品能力写得很清楚：门店可创建非油SKU、C端下单详情页出现“随手购”模块、多SKU加购、油+非油合单支付、结算环节复用现有订单/账单/小票能力，这些都是明天介绍商户时要讲的人话版本。
- BRD和“从价值到落地的主线”已经给出收益模型：基于油单量 × 非油GMV渗透 × 净费率，年利润目标约2,000万、GMV渗透目标5%，还有明确的L1高频低风险SKU白名单（瓶装水、纸巾、雨刮水…). 这可以直接改成“你这个站，如果一天××单油，我们帮你多挣多少钱”的算式。
- 对商户来说，他们要的不是“平台大叙事”，而是三句话：① 我用这套东西能多赚多少？② 要多花多少人/钱/时间？③ 风险和麻烦有多大（投诉、对账、发票、履约责任）？你手上的BRD/PRD已经给了答案，你要做的是翻译成他们能听懂的商业语言和例子。

**（三）落地层面，你需要准备好：材料、问题、分工和会后动作**

- 材料上：至少要有一页纸产品介绍、简易收益测算模板、C端/B端流程图或Demo截图、初版合作模式和政策要点（费率区间、账期、准入门槛、试点支持）、以及一份“访谈记录表”。这些内容在PRD/BRD和主线文档里都有结构，只需要抽取和重排。
- 过程上：要用结构化访谈，而不是闲聊：从“非油现状 → 线上经验 → 对随手购兴趣/顾虑 → 履约能力 → 合作条件期望”一条线问下来，每一块都要有1–3个关键问题；区域同学要提前分工：谁主讲、谁问、谁记、谁专门观察现场动线/货架/人手。
- 会后上：每个站至少要拿到四个结果：① 是否有兴趣进入样站白名单（强烈/一般/保留）；② 一位明确的项目对接人和联系方式；③ 一份初步的SKU清单和预估日单；④ 一个双方都认可的“下一步动作和时间点”（例如“下周给你发收益测算+样站方案”，“两周内完成开户和培训”）。这些，才是明天拜访的硬产出。

### 3）思维导图结构（文本分层）

#### 一、拜访目标对齐

1. 明确本次拜访要达成的三件事  
	1.1 验证商户对“非油随手购”的兴趣与接受度  
	1.2 评估商户履约能力和资源投入意愿  
	1.3 收集商户对合作模式（费率/账期/物料/流量）的预期与顾虑
2. 产出物定义  
	2.1 访谈记录表（结构化信息）  
	2.2 意向站点清单（强/中/弱意向）  
	2.3 后续跟进计划（每站一个下一步动作）

#### 二、拜访前：资料和工具准备清单

1. 产品/方案材料  
	1.1 《随手购是什么》一页纸
	- 用户侧价值：不断车、10分钟达、合并支付
	- 商户侧价值：提升非油营收、作业数字化、复购沉淀  
		1.2 C端体验截图或Demo
	- 下单详情页“随手购”模块
	- 确认订单页（油+非油合单）
	- 完单页与订单详情页的非油金额展示  
		1.3 B端/站点作业流程示意
	- 接单→拣货→贴码→车边核验→回执→对账/结算
	- 对应10分钟SOP关键时间拆解
2. 商业测算小工具  
	2.1 简易Excel/纸质表：
	- 每日油单数D
	- 预估非油GMV渗透率s（如3%/5%）
	- 非油客单价AOV\_non-oil（如30–60）
	- 平台服务费率r（如2–3%）  
		2.2 算法展示：
	- 年非油GMV ≈ D × s × AOV\_non-oil × 365
	- 平台服务费/商户毛利示例
3. 合作模式/政策要点草稿  
	3.1 准入条件
	- 有便利店/有非油货架/有发票能力
	- 人手、库存、动线、系统、合规评分≥70（可口头简化表达）  
		3.2 商业条款方向
	- 费率区间（示例：2–5%，与SLA/规模挂钩）
	- 账期（示例：T+7为默认，试点期可以更灵活）
	- 支持项（物料、培训、运营支持、线上资源位）
4. 访谈与记录工具  
	4.1 打印版/电子版《商户访谈单》  
	4.2 笔+录音设备（如对方同意）  
	4.3 区域同学分工表（主讲/记录/观察）

#### 三、拜访中：访谈提纲与关键问题

1. 开场与破冰（2–5分钟）  
	1.1 简单自我介绍+项目一句话定位
	- “我们想用加油这个高频流量，帮你把便利店/车品再多卖一块钱.”  
		1.2 拜访目的说明
	- 了解您现在做非油的情况
	- 看看这套“随手购”方案能不能帮你多赚
2. 了解非油现状（5–10分钟）  
	2.1 当前非油经营情况
	- 日均非油流水、主打品类、毛利高的品类
	- 是否已经在美团/饿了么/其他线上平台卖非油  
		2.2 难点和痛点
	- 人手是否够用？哪个时段最忙？
	- 动线是否方便到车边交付？
	- 对账、发票、退款有没有觉得麻烦的地方？
3. 介绍随手购产品（5–10分钟）  
	3.1 从用户体验视角讲一遍
	- 加油时在APP里随手勾选水/纸巾/车品 → 一起付款 → 加完油直接拿货走  
		3.2 从站点作业视角讲一遍
	- 站内手机/终端接单 → 按清单拣货→贴码→送到车边→确认交付 → 系统自动记账  
		3.3 强调边界
	- 目前只做高频低风险SKU（瓶装水、纸巾、雨刮水等），不碰冷链/鲜食/药械/危化
4. 收集商户对方案的第一轮反馈（10分钟）  
	4.1 兴趣与顾虑
	- 你觉得这样的方式对你的生意有没有帮助？
	- 最担心的地方是什么？人手不够？投诉？系统不会用？  
		4.2 履约能力与SOP可行性
	- 忙时/闲时人手配置？
	- 从便利店到加油车位大概多久？
	- 你觉得“10分钟内交付”这个要求，在哪些时段/天气可能会有困难？
5. 谈合作模式和预期（10分钟）  
	5.1 用测算小工具算一算
	- 以他站当日油单数试算一年能多出多少非油营收
	- 平台抽成、他的毛利分别大致是多少  
		5.2 问清他的底线/期望
	- 能接受的大致费率范围？
	- 能接受的账期？现在其他平台怎么结？
	- 对物料/培训/线上流量有什么期待？
6. 收尾与下一步（5分钟）  
	6.1 确认意向等级（强/中/弱）  
	6.2 确定站点对接人+联系方式  
	6.3 约定下一步
	- 什么时候把站点方案/收益测算表发给对方
	- 预期何时可以启动试点（如：春节前/黄金周后）

#### 四、拜访后：沉淀与跟进

1. 数据和信息汇总  
	1.1 每个站点的访谈记录结构化录入  
	1.2 输出“站点画像+意向等级+风险点”
2. 决策和优先级  
	2.1 结合BRD中的选站评分维度（人手/库存/动线/系统/合规）给站点评分  
	2.2 形成首批样站推荐名单
3. 内部对齐与方案迭代  
	3.1 和区域/运营/产研同步访谈结果  
	3.2 迭代招商政策、SOP细节和物料设计  
	3.3 为下一轮拜访或签约准备更有针对性的版本

### 2）三段论据（现状与问题 → 关键洞察 → 路径方案）

**（一）现在最大的问题，是信息都在聊天里，无法沉淀成“可比较、可决策”的数据**

- 你这次拜访的目标，不是单纯说服商户签约，而是“打捞一手事实”：非油怎么卖、线下物料怎么投、什么产品真在动、对服务费的底线和顾虑是什么。
- 如果只靠自由聊天+会后回忆，很容易出现信息缺失、不同拜访人记录风格不一，后续根本无法横向对比“哪类站更合适做样板”。
- 因此，必须有一份统一模板，把关键信息收敛在可勾选/可打分/可简述的字段里，做到“不同区域、不同拜访人填出来的东西可以放一张表里对比”。

**（二）线下物料、爆款打造、服务费接受度，这三块是决定能不能规模复制的关键变量**

- 线下物料：决定你能不能在站内把“随手购”心智种下去——目前站点物料多为品牌投放、油品促销，对“非油线上化”心智基本空白，这块调研清楚，后面才能做标准化物料和A/B测试。
- 爆款产品：随手购要跑得通，必须有清晰的“站点爆款组合”，而这个不在PPT里，在货架和收银台上——访谈表里必须强制商户说出“现在卖得好/毛利高/适合车主随手买”的SKU。
- 服务费接受度：这是招商能不能成交的“终极问题”，你需要拿到商户对不同费率+资源组合的真实反馈，而不是只记录一句“觉得费率有点高”。

**（三）用“模块化访谈表 + 半结构化记录方式”，既方便现场使用，又方便后续分析**

- 按模块拆：基础信息、非油现状与诉求、线下物料与动线、爆款&组合、随手购方案反馈、履约能力、服务费接受度、意向评级&下一步。每个模块3–7个关键字段，控制在一页纸正反可手写完成。
- 记录方式上，使用“勾选 + 数值 + 简短原话”混合：既不增加拜访负担，又能保留商户的关键表述（例如对费率的原话）。
- 会后你可以直接把这些访谈表录成结构化信息，用来：筛选首批样站、沉淀不同类型站点画像、迭代爆款池和物料策略、优化服务费政策的档位和话术。

### 1）一句话结论

商户版招商一页纸，要用“这是什么 → 能多赚多少钱 → 要做什么 → 怎么分钱 → 我们给什么支持”这一条线，把随手购讲得又简单又算得清，让商户当场有兴趣、回去愿意试。

### 2）三段论据（现状与问题 → 关键洞察 → 路径方案）

**（一）现在的痛点：商户听不懂产品，只关心两件事——钱和麻烦**

- 站长/店长没有精力听复杂的产品逻辑，他们只关心：① 一年能多赚多少钱；② 会不会给自己找麻烦（投诉、对账、人手）。
- 如果一页纸堆功能、堆平台视角，商户只会觉得“又是一个平台要抽成”，不会把随手购当成真实生意机会。
- 所以，一页纸必须从商户视角出发，用他们的语言聚焦四个问题：能多赚、少麻烦、门槛低、支持足。

**（二）你的优势：非油随手购天然“顺带赚”，只要说清楚算式和边界**

- 相比外卖/即时零售，随手购其实是“顺路成交”：客流本来就有、车就在油机旁、货就在便利店里，商户不需要重新获客，只是把现有货更高效卖一遍。
- 平台在系统、流量、选品、物料上的投入，本质是在帮商户把“加油一次，多赚一单”的链路打通；只要算式简单、服务费透明，商户可以接受合理抽成。
- 你可以用一个具体站点的测算例子，把复杂的GMV、渗透率、毛利率，转成“每天多卖几箱水、几包纸，一年多出几万块净利润”这种话。

**（三）一页纸的结构：上半页抓注意力，下半页给算式和落地路径**

- 上半页：标题 + 金句 + 3 个核心卖点 + 一小块“收益示例”。目标是让站长在 10 秒内理解“这是帮我在加油时顺便多赚钱的工具，不需要大改造”。
- 下半页：合作模式说明（服务费/账期/边界）+“你需要做的 3 件事”+“平台提供的支持”+ 试点名额/联系人，把“不确定感”压下去，让商户觉得“可以先试一试”。
- 你这版一页纸，可以先做成“无价格敏感信息的通用模板”，后续再按区域/商户类型微调服务费文案和爆款示例。

**【模块二：能多赚多少钱（示例测算）】**

> 小标题： **一天 600 单加油，保守 3% 用户随手买，一年多赚 4,000+ 元净利润起步（还不含被带动的线下消费）**

示例假设（可按站点实际数字现场改口径）：

- 日均加油订单数： **600 单**
- 随手购非油渗透率： **3%** （即每 100 位加油顾客，有 3 位顺带买一单）
- 非油客单价： **30 元** （矿泉水+纸巾/小零食等组合）
- 非油毛利率： **30%**
- 平台服务费（示例）： **3%**

简单算式：

1. 日非油 GMV ≈ 600 × 3% × 30 ≈ **540 元/天**
2. 年非油 GMV（按 30 天/月）≈ 540 × 30 ≈ **16,200 元/年**
3. 商户毛利 ≈ 16,200 × 30% ≈ **4,860 元/年**
4. 平台服务费 ≈ 16,200 × 3% ≈ **486 元/年**
5. 商户净收益 ≈ 4,860 – 486 ≈ **4,374 元/年**

可以这样给商户落地话术：

> “就按您这站一天 600 单油，只要有 3% 的人顺手买一单水+纸，一年就有 1.6 万左右的成交额，扣掉服务费，您还多赚 4,000 多块净利润；如果渗透做到 5%，收益直接往上翻。”

### 1）一句话结论

用户调研“值得做，而且必须做”，但不能做成大而全研究项目，而要围绕随手购的关键决策假设，做一轮轻量、可复用的用户洞察，把它变成：一方面支撑你们BC两端的营销策略，另一方面反过来当成对商户的“价值承诺”和招商抓手。

### 2）三段论据（现状与问题 → 关键洞察 → 路径方案）

#### （一）现状与问题：现在所有 BC 叙事都停留在“想象中的车主”，缺乏可以拿得出手的证据

- 你现在对 C 端的理解，更多来自常识和内部假设：车主高频来加油、在车里有碎片时间、对瓶装水/纸巾/车品有即买即用需求……但这些都没有被“用户自己的嘴”和真实行为验证。
- 对 B 端（商户）来说，他们天然会质疑：“车主真的会在手机上点这些东西吗？”、“你说 3% 渗透率是拍脑袋还是真数据？”——缺少用户调研的支撑，你在招商时只能用“我们判断”“我们预计”，说服力有限。
- 对你自己来说，没有用户调研，你很难回答几个关键问题：
	- 车主到底在什么场景/心情下愿意点随手购？
	- 他们更在意“省事/便宜/品质/品牌/即时可用”中的哪几个？
	- 他们对“车边交付时效”“价格溢价”“组合搭配”的容忍度在哪里？  
		没有这些答案，BC 两端的营销策略都只能做“广撒网试试”。

#### （二）关键洞察：用户调研本质上是“帮商户和平台少踩坑”，是非常好用的招商筹码

- 商户真正想听到的不是“我们觉得车主会喜欢”，而是“我们已经调研过 ×× 位车主，他们说……因此我们帮你选了这几款爆款+这个价位，是有依据的”。这会把你从“卖产品的人”变成“懂车主、懂运营的伙伴”。
- 如果你能在招商沟通中说清楚：
	- “我们会先在首批样站上做一轮用户调研，帮你搞清楚你站车主的真实偏好，之后选品和活动都基于这些洞察来设计。”  
		这对商户是有吸引力的——因为绝大多数组合油站/便利店自己是没能力做用户研究的。你等于在说：“我们带着研究能力和方法来帮你把生意看懂”。
- 从平台角度看，用户调研并不是“额外开销”，而是给后面所有动作铺底：
	- 随手购的 UI 文案、入口位置、推荐逻辑
	- 线下物料的主诉求（省事？安全？便宜？好货？）
	- 不同城市/路况/时间段的爆款组合（通勤 vs 长途 vs 家用车 vs 网约）  
		这些一旦踩对了，你的 GMV 复利效应会远高于调研的成本。

#### （三）路径方案：做一轮“轻量却锋利”的 MVP 用户调研，而不是冗长的大课题

我会建议你这样处理：

1. **明确调研定位：只回答 5–7 个“对生意有直接影响”的核心问题**
	- 比如：  
		1）车主在加油时，愿不愿意在手机上点非油？阻碍是什么？  
		2）车主对“车不下、人不走、10分钟送到车边”这套价值，哪个点最打动？  
		3）在你们白名单的品类里，车主最倾向买什么？接受什么价位？  
		4）车主更能接受哪种优惠方式：满减 / 加价购 / 第二件半价 / 直降？  
		5）他们对“比线下货架贵一点但送到车边”的接受度如何？  
		6）对履约时效和交付方式的容忍边界（超 10 分钟是否介意、愿不愿意自取等）。
	- 所有问题都必须能回落到：选品、定价、营销话术、物料内容、SOP设定。不能答的就别问。
2. **用“三板斧”做 MVP 调研组合：线上轻量问卷 + 站点拦访 + 行为数据搭配**
	- 线上问卷（轻）：
		- 在现有 APP / 小程序 / 加油后短信中嵌一个 5–8 题的极简问卷（必要时配小额积分/优惠券），获取基础偏好、价格敏感度、对随手购概念的接受度。
	- 站点拦访（深）：
		- 在首批意向站点，挑高峰+平峰各几个时段，每站找 10–15 个车主做短访（5 分钟内），用统一话术问关键问题，顺便观察他们对线下物料/入口提示的反应。
	- 行为数据（实）：
		- 随手购上线后，抽样看：曝光→点击→加入→下单→履约的完整漏斗，和不同商品/套餐的点击率、转化率。  
			三者结合，能互相校验：问卷说想买的东西，实际会不会点；拦访说怕麻烦，数据里看是不是时效抖一下就掉单。
3. **从一开始就把“用户调研”设计成一个招商故事，而不是内部研究项目**
	- 对外给商户的说法可以结构化为：
		- 我们会为首批样站提供一套“用户洞察 + 爆款建议”的服务，包括：  
			1）样站车主轮廓分析（谁在来、什么时候来、典型画像）  
			2）样站车主的随手购需求和价格带偏好  
			3）基于洞察的专属爆款组合建议 + 活动建议
		- 你可以用一句话去讲：
			- “您把站点开出来、货备好，我们帮您把车主看懂，然后再一起把货卖出去。”
	- 内部产出物上，至少要沉淀三类：  
		1）《非油随手购车主洞察小册子》（5–10 页，招商、培训共用）  
		2）不同类型站点/城市的“爆款组合示例卡”（可直接给商户看）  
		3）线下物料主诉求与文案规范（比如到底强调“不断车”还是“更便宜”）。
	- 这会极大增强你在后续招商中的话语权：
		- 先招商一批愿意试的 → 用他们的数据+用户调研做洞察 → 带着洞察和样板站回头去打更大的站和区域。

#### 五、线下站点拦访提纲（5 分钟版）

> 线下不宜照搬线上问卷，而是用 5 个主问题 + 若干追问，获取“真实语境下的原话”和行为细节。

1. 问题 1：
	- 主问：您平时加油时，手机一般在干嘛？
	- 追问：
		- 会不会顺带看一眼 APP 上有什么优惠？
		- 有没有因为看手机错过过什么提示？
2. 问题 2：
	- 主问：您会不会顺便在油站买水、纸巾之类？为什么？
	- 追问：
		- 什么情况下买？（比如天气热、家里没水了等）
		- 什么情况下明明想买但放弃？
3. 问题 3（概念测试）：
	- 主问：如果加油的时候，在手机上可以顺手点几样东西，付完款，等加完油工作人员把东西送到您车边，您愿意试吗？
	- 追问：
		- 哪一点最有吸引力？哪一点最让你担心？
		- 你希望上面有哪些商品？大概要花多少钱？
4. 问题 4（时效与体验）：
	- 主问：您觉得从点完单到拿到货，大概几分钟比较合适？
	- 追问：
		- 如果忙一点，慢 3–5 分钟，您接受吗？
		- 您更希望是他来找您，还是您自己去拿？
5. 问题 5（价格与优惠）：
	- 主问：如果随手购里的东西比超市贵一点，但省得下车，您能接受贵多少？
	- 追问：
		- 您更在意“优惠力度”还是“省事”？
		- 哪种活动您最容易被打动？（满减/加价购/第二件…）

> 所有拦访都要求记录关键原话 + 性别/大致年龄/车用途，方便后续形成“用户画像 + 语录卡片”，用于招商和物料文案。
