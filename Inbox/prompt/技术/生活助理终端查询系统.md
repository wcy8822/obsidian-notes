# 生活助理终端查询系统：信息管理与命令交互框架（适配终端调用）
核心目标是打造「本地信息结构化存储 + 终端命令快速查询」的生活助理，让你通过简洁终端指令（如`查询 结婚纪念日`、`获取 公司VPN网址`），即可精准调取关键信息，无需手动查找文件。以下是从「信息结构化」到「终端命令交互」的完整框架，确保AI严格按规则执行。


## 一、第一步：生活信息结构化（终端查询的基础）
先将你的生活信息按「分类-字段」标准化，确保AI能精准匹配命令与信息。建议用 **Excel/CSV/TXT** 记录，格式如下（可根据需求扩展分类）：

### 1. 信息分类与必填字段（AI需严格按此解析）
| 信息分类       | 核心字段（必填）                | 可选字段                | 示例                          |
|----------------|---------------------------------|-------------------------|-------------------------------|
| 生活记忆       | 记忆标签（唯一标识）、具体内容  | 关联人/日期/备注        | 标签：结婚纪念日；内容：2020.10.01 |
| 常用工具       | 工具名称、访问地址/路径/指令    | 账号/使用场景/备注      | 名称：公司官网；地址：https://xxx.com |
| 账户密钥       | 账户/API名称、密钥值            | 有效期/使用范围/备注    | 名称：OpenAI API；密钥：sk-xxx |
| 日程提醒       | 提醒标签、时间、具体事项        | 重复频率/关联人/备注    | 标签：每周五会议；时间：14:00；事项：部门周会 |
| 地址通讯       | 地址标签、详细地址              | 联系人/电话/导航链接    | 标签：家里地址；地址：XX市XX区XX路 |


### 2. 信息录入约束（需告知AI）
- 「记忆标签/工具名称/账户名称」必须**唯一且简洁**（如“家里WiFi密码”而非“WiFi”，避免命令匹配混淆）；
- 敏感信息（如密钥、密码）需标注「敏感项」，终端查询时直接返回原始值（无需脱敏）；
- 日期/时间统一格式（如“2024-05-20”“09:30”），避免AI解析错误。


## 二、第二步：终端命令设计（简洁、直观，适配手动输入）
设计标准化命令格式，让你在终端输入时无需复杂语法，AI能快速解析意图。命令分为「核心操作+关键词」两部分，支持3类核心场景：


### 1. 命令格式规范（AI需严格识别）
| 操作类型 | 命令格式                | 示例                          | AI解析逻辑                          |
|----------|-------------------------|-------------------------------|-------------------------------------|
| 精准查询 | `查询 [关键词]`         | `查询 结婚纪念日`              | 匹配「信息分类」中的「唯一标识」（如“结婚纪念日”→生活记忆），返回「具体内容」 |
| 获取详情 | `获取 [关键词] [字段]`  | `获取 公司官网 地址`          | 先匹配关键词“公司官网”，再返回“访问地址”字段的值 |
| 模糊查询 | `查找 [关键词片段]`     | `查找 WiFi`                   | 匹配所有包含“WiFi”的标识（如“家里WiFi密码”“办公室WiFi名称”），列出所有结果供选择 |


### 2. 命令约束（避免AI误解）
- 关键词优先匹配「唯一标识」（如“查询 OpenAI”→优先匹配“OpenAI API”），不匹配非标识字段（如不匹配“备注”中的内容）；
- 若关键词对应多个结果（如“查找 银行卡”→“工资卡卡号”“信用卡账单日”），AI需按「分类+标识+核心字段」列表展示，不遗漏；
- 终端返回结果需「极简」：仅输出目标信息，不添加额外说明（如查询结婚纪念日→直接返回“2020.10.01”，不附加“你的结婚纪念日是”）。


## 三、第三步：AI交互流程（从“信息读取”到“终端响应”）
AI需按「读取信息→生成配置→解析命令→返回结果」四步执行，每步有明确约束，确保不偏离目标。


### 1. 步骤1：读取本地信息，生成“查询配置库”
你将本地文件（Excel/CSV）提供给AI后，AI需执行：
1. **解析核对**：按「信息分类」统计条数（如“生活记忆：8条，常用工具：12条”），向你确认无遗漏；
2. **生成配置**：将信息转为「JSON格式配置库」（终端查询的底层数据），结构如下（AI需严格按此生成）：
   ```json
   {
     "生活记忆": [
       {"标识": "结婚纪念日", "内容": "2020.10.01", "关联人": "配偶", "备注": "每年休假庆祝"},
       {"标识": "孩子生日", "内容": "2023.05.15", "关联人": "孩子", "备注": "需准备礼物"}
     ],
     "常用工具": [
       {"标识": "公司官网", "地址": "https://xxx.com", "备注": "需VPN访问"},
       {"标识": "常用文档库", "地址": "D:/工作/常用文档", "备注": "本地路径"}
     ],
     "账户密钥": [
       {"标识": "OpenAI API", "密钥": "sk-xxxxxxxxxx", "有效期": "2025-12-31", "备注": "用于AI开发"}
     ]
   }
   ```
3. **确认保存**：提示你“配置库已生成，可保存为本地文件（如`life_assistant_config.json`），后续查询基于此文件”。


### 2. 步骤2：终端命令解析与响应（核心交互逻辑）
当你在终端输入命令（如`查询 公司官网 地址`），AI需按以下流程响应：
1. **命令拆解**：提取「操作类型=获取」「关键词=公司官网」「字段=地址」；
2. **信息匹配**：在配置库中找到“常用工具”分类下，“标识=公司官网”的条目；
3. **结果返回**：终端输出极简结果→`https://xxx.com`（无多余文字）；
4. **异常处理**：若出现以下情况，按规则提示：
   - 无匹配结果：输出`未找到关键词“XX”的信息，请检查关键词或补充到配置库`；
   - 关键词模糊：输出`找到2条匹配结果：1. [分类] 标识1：核心内容；2. [分类] 标识2：核心内容，请补充关键词`；
   - 字段缺失：输出`“XX”标识下无“XX字段”，请检查配置库是否补充该字段`。


### 3. 步骤3：信息更新与维护（支持终端命令扩展）
若你需要新增/修改信息，可通过终端命令触发，AI需支持：
- 新增信息：命令`新增 [分类] [标识] [内容]`（如`新增 生活记忆 父母生日 2024-06-20`），AI自动更新配置库，并提示“已新增，配置库已同步”；
- 修改信息：命令`修改 [标识] [字段] [新内容]`（如`修改 OpenAI API 密钥 sk-yyyyyy`），AI覆盖旧值，提示“已修改，建议备份新配置库”。


## 四、第四步：AI约束指令（确保不遗忘、不偏离）
将以下指令固定告知AI，每次交互前可重申，避免AI忽略规则：
```
你是“生活助理终端查询小能手”，需严格遵循以下规则：
1. 仅基于我提供的「life_assistant_config.json」配置库响应，不编造任何信息（即使你“知道”通用内容，也以配置库为准）；
2. 终端命令仅识别“查询/获取/查找/新增/修改”5类操作，按“操作+关键词”解析，不处理无关命令；
3. 返回结果必须极简：仅输出目标信息，不添加问候语、解释性文字（如查询结婚纪念日→直接返回日期，不写“你的结婚纪念日是”）；
4. 敏感信息（密钥、密码）直接返回原始值，不脱敏；
5. 每次响应后，仅附加1句提示：“是否需要查询其他信息？”，不主动扩展话题；
6. 若我未明确命令，仅提示“请按格式输入命令：查询/获取/查找 [关键词]”，不引导其他操作。

当前配置库已提供，现在请等待我的终端命令，按规则响应。
```

通过这套框架，你只需：1. 按规范整理本地生活信息；2. 用简单终端命令查询；AI则会严格按“配置库匹配+极简响应”执行，成为你的高效生活查询助手，无需反复查找文件或记忆复杂信息。
