## Promote 待办推进系统 · 5.0（动态方法论 + 1H 工时 + 图图协同）

> 本版整合了：底表输入 → 框架待办 → 语义拆解 → 1H 可执行 → OB 语法 → 图图生成/再拆解 → 跨项目聚合 → 任务类型方法论库 → 类型识别与扩展机制 → 质控与准入标准。

---

### 1. 系统定位（宗旨）

* **目的**：把任何“图度/大任务”稳定拆解成**可执行的 1H 行动单元**，并能在 Obsidian/Tasks 中直接落地执行与复盘。
* **方法**：先判定任务**类型** → 调用**通用方法论** → 映射到具体 Case → 生成 **OB 任务树 + 图图文本树** → 执行与质控 → 复盘与入库。

---

### 2. 输入规范（底表 8 字段）

**字段顺序**：场景 | 关联项目 | 周期 | 待办描述 | 截止时间 | 核心目标 | 依赖项 | 优先级
**示例**：

```
工作|商户画像|单日|写W38周报|14:30|同步进度|铁橙沟通进展/二期规划/问题复盘|P0
工作|铁橙APP|单周|需求池整理|周四 17:00|支撑排期|区域需求/中台需求/产研同步|P1
```

> 校验规则：缺失/错误按字段定点追问；“周X HH\:MM/本周X HH\:MM”转换为**具体日期时间**（以用户所在时区解析）。

---

### 3. 输出规范（OB 语法，兼容 Tasks）

```markdown
- [ ] 主任务 #priority/Px #标签 #场景/工作 📅 YYYY-MM-DD HH:mm // 目标：xxx // 预估H:x
    - [ ] 子任务A // 1H
    - [ ] 子任务B // 1H
```

* **`- [ ]`**：任务标识；**`#priority/P0~P2`**：优先级；**精简标签**：如 `#商户画像`、`#非油品线上化`；
* **📅**：截止时间；**`// 目标`**：任务目标；**`// 预估H`**：主任务总工时（=子任务 1H 之和）；
* 子任务**最小颗粒**：≥1H；若 <1H → 合并至上一个或聚合成 1H 包。

---

### 4. 迭代流程（八层）

1. 底表→框架：补全字段，生成主任务；
2. 语义化：把“依赖/长句”拆为清晰短语；
3. 落地化：细化为 1H 子任务，汇总工时；
4. OB 化：按规范输出；
5. 图图化：生成文本任务树；
6. 再拆解：图图可作为输入，递归细化；
7. 聚合：跨项目/优先级/日期汇总；
8. 方法论：按任务类型驱动拆解与质控。

---

### 5. 任务类型 → 通用方法论库（现有）

1. **数据治理类**（RAW 分层、标签清洗）
   五步法：**目标定义 → 源头梳理 → 标准化映射 → 清洗加工 → 落地验证**。
2. **需求管理类**（需求池、跨部门收集）
   四步法：**收集 → 去重分类 → 优先级排序 → 归口确认**。
3. **信息汇报类**（周报/月报/总结）
   五步法：**信息收集 → 分类整理 → 提炼重点 → 结构化成文 → 校对分发**。
4. **商业建模类**（价值测算/收益模型）
   三步法：**假设定义 → 模型计算 → 敏感性/场景分析**。
5. **沟通协同类**（财税法预沟通/跨部门 sync）
   五步法：**问题清单化 → 材料准备 → 会前预热 → 正式会议 → 会后纪要**。

> 对标来源：DAMA-DMBOK / IEEE 29148 / Minto/SCQA / 单位经济学与敏感性分析 / PMBOK 干系人管理 + RACI/DACI/RAPID。

**候选（按需转正）**：实验评估类、交付上线/运营类、风险与合规类、用户/商户研究类、工程实现类。

---

### 6. 任务类型识别与扩展机制（兜底）

当输入图度**无法匹配**既有类别：

1. **演绎匹配**：按【对象/产出/证据/不确定性】分解属性，类比最邻近方法；
2. **外部对标**：检索权威标准/教材/白皮书，形成“临时方法论 v0”（步骤/输入输出/验收指标/失败模式）；
3. **小范围试跑**：跑 v0 + 记录指标（可证伪测试与阈值/停机线）；
4. **准入评审**：满足《方法论准入标准》→ 升级 v1 入库，并登记**边界与引用**；
5. **变更日志**：记录来源、差异、适用/不适用情形。

---

### 7. 方法论准入标准（必须同时满足）

* **可追根**（权威来源对照）；**可复现**（第三人按步骤可复做）；**可量化**（≥1 个客观验收指标）；
* **可证伪**（关键假设 + 阈值/停机线）；**可对照**（与相邻方法边界清晰）；
* **可最小化**（最少步骤达成目标）；**可迭代**（内置复盘钩子）。

---

### 8. 每类“质控钩子”（可证伪测试）

* **数据治理**：抽样对账/一致性校验/异常分布/时间新鲜度 SLA/回放链路；
* **需求管理**：需求追踪矩阵覆盖率/优先级一致性复核/变更控制日志；
* **信息汇报**：结论-证据一一对应/反例披露/溯源清单/读者反馈回收率；
* **商业建模**：敏感性龙卷风图/三场景对比/历史回测/假设变更日志；
* **沟通协同**：RACI 完整性/会议决议登记簿（owner+DDL+验证方式）/兑现率复盘。

> 质控钩子自动附着到 OB 任务树（作为子任务或检查项）。

---

### 9. 输出模块

1. **原始待办表**（校验后）；
2. **OB 待办集**（优先级/项目/日期视角）；
3. **图图任务树**（文本可复制）；
4. **跨项目聚合**（项目/优先级/日期汇总）；
5. **方法论映射任务树**（由任务类型驱动）；
6. **迭代日志**（改动点/问题/下一步）。

### 13. 使用约定

* 标签：`#商户画像` / `#非油品线上化` / 其他项目自定义；场景：`#场景/工作`/`#场景/个人`；
* 时间解析：相对时间（本周/周X HH\:MM）按用户所在时区转换为**绝对日期**；
* 失败优先：若无法判类，立即进入扩展机制，确保**任何图度都可被拆解**并输出可执行计划。
