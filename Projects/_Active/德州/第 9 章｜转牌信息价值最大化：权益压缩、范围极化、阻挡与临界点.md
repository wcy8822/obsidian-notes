## 9.1 一句话结论

**转牌不是“再打一枪看看”，而是“权益开始压缩、范围开始极化、阻挡价值真正放大的临界点”：你在这里做的每一个 bet / check / fold，都会比 flop 贵得多，也更接近整手牌的最终 EV。**

---

## 9.2 章节目的：Turn 这一街到底要解决什么问题？

这一章要让你，在看到 turn 牌时，脑子里自动跑完四个问题：

1. **权益压缩（equity compression）：**
    
    - 这张 turn 牌，让双方整体 equity 分布发生了什么变化？
        
    - 是否出现了“优势进一步放大”或“原本劣势一方突然翻盘”的结构性变化？
        
2. **范围极化（range polarization）：**
    
    - 我这一条线（bet / check-back / XR）在 turn 是否应该从“合并范围（merged）”切换为“极化范围（nuts + bluff）”？
        
    - 是否需要使用更大的 size（2/3–超池）来放大极化的价值？
        
3. **阻挡（blockers）：**
    
    - 在 turn 继续下注 / 加注 / 抓诈时，我手牌对对手 value / bluff 组合的阻挡作用如何？
        
    - 是“好阻挡”（少阻 value，多阻 bluff），还是“自废武功”的坏阻挡？
        
4. **临界点（threshold）：**
    
    - 当前底池与尺⼨下，我的 bluff 需要对手弃多少才不亏？
        
    - 我 call 需要多少 equity 才不亏？
        
    - 我这手牌距离这些临界点有多远？
        

本章只打基础逻辑和框架：

- 不细讲“double barrel 频率”（那是第 10 章的任务），
    
- 只讲：**你在 turn 怎么看牌面与范围，让后面的 barrel/放弃变成“顺水推舟”，而不是纯情绪决策。**

## 9.3 GTO 基线一：Turn 为何“信息价值最大”？（Janda 视角）

Janda 在《进阶指南 (2)》里面有一条很关键的观念：

> **转牌是“信息量 × 底池大小”的乘积最大的一街。**

拆一下这句话：

1. **翻牌信息多，但底池还小**
    
    - flop 第一次摊 3 张公共牌，信息增量最大；
        
    - 但 pot 通常只是 1–2 个下注后的体量，任何轻微失误，后面还有修正空间（check-back/便宜摊牌等）。
        
2. **河牌底池最大，但信息增量有限**
    
    - river 只再摊一张牌，很多范围关系在 turn 已经定型；
        
    - 你更多是在完成既定的极化线，而不是“重新发明轮子”。
        
3. **转牌两头都大：**
    
    - pot 已经经历至少一轮下注（flop），体量显著放大；
        
    - 又摊出了一张关键公共牌，
        
        - 要么巩固原有优势（blank）；
            
        - 要么彻底翻盘（scare card / range shift）。
            

**结论：**

- turn 是“你要为整手牌做最终规划”的一街：
    
    - 决定这条线是否进入极化模式（未来能否打到 all-in）；
        
    - 决定哪些牌成为 river bluff / bluffcatch 的候选；
        
    - 决定哪些牌在这里就砍掉，承认 EV≈0。
        

**你如果在 flop 已经有完整思路，那么 turn 是你“兑现思路”的地方；  
你如果 flop 是 autopilot，turn 就会变成“补锅现场”。**

## 9.4 GTO 基线二：从 flop 到 turn 的“权益压缩”

《全范围 C-Bet 指南 II》大量对比了：

- UTG vs BB / BTN 在各种 flop 上的 **整体范围权益**（用箱型图 + 中位数线展示），
    
- 并且设定了一个固定策略：
    
    - flop 统一用 30% pot cbet，
        
    - turn & river 统一用 70% pot 下注。
        

在这些报告里，你可以看到一个共性现象：

1. **flop：整体权益差距“看起来不大”**
    
    - 很多 UTG vs BB 的牌面，UTG equity 可能只是 55% vs 45%；
        
    - 很多 BTN vs BB 的牌面，BTN equity 也就 53–57% 范围。
        
2. **turn：权益开始“压缩”**
    
    - 一部分牌型在 turn 彻底变成“接近 0%”的垃圾（完全 miss + 无后门）；
        
    - 一部分牌型在 turn 变成“接近 80–90%”的强牌（成顺 / 成同花 / 成两对）。
        
3. **对整体范围而言：**
    
    - flop ：大量组合的 equity 介于 30–70%；
        
    - turn ：更多组合靠近两端（<20% 或 >80%）；
        
    - 这就是我们说的 **“权益压缩”**。
        

**为什么你要关心这个？**

- 因为这意味着：
    
    1. flop 很多牌可以“勉强继续看一张”；
        
    2. turn 很多牌必须决定“要么转身离场，要么准备迎接大底池”；
        
    3. 再拖到 river 才“想起要做决策”，通常已经太迟、太贵。
        

因此，**turn 是你主动利用权益压缩来极化范围、清除垃圾、保留候选的主战场。**

## 9.5 来自《超池下注指南》的典型 turn 极化结构

《超池下注指南》给了几个非常典型的 turn 极化例子，这里抽象出两类你之后会反复用到的结构。

### 9.5.1 结构 A：IP flop 小注 → turn blank → IP 超池极化

结构示意（SRP / 3BP 都常见）：

- flop：
    
    - IP 在自己明显范围优势的牌面（如 A 高干面）用 **30% pot 高频 cbet**；
        
    - BB 用较宽范围继续（大量中对 / 后门牌 / 高牌）。
        
- turn：
    
    - 摊出 blank（对子 / 完全无关小牌）；
        
    - 结果是：
        
        - IP 范围仍然“上端厚 + 中下端一堆空气”；
            
        - BB 范围被强烈封顶（cap）：缺乏真正 nuts，很多是中等 pair / weak draw。
            

**solver 给出的答案：**

- IP 在这种结构下，
    
    - 把 turn 尺度切换到 **70% pot 或超池（100–150%）**；
        
    - 范围从 flop 的“较为合并”切换为 **极化**：
        
        - 顶端：强 value（两对+ / top set / top two / nut draw）；
            
        - 底端：强听牌 + 合理阻挡的空气。
            

**权益压缩 + 范围封顶的组合，给了 IP 足够理由“在 turn 兑现极化价值”。**

### 9.5.2 结构 B：BB flop 跟 IP 超池 → turn blank → BB 超池反击

另一个在《超池下注指南》里出现的经典场景：

- flop：
    
    - IP 在某些 AKQ / KQTs 这类 nuts 极端偏 IP 牌面上，用 **150% pot 超池下注**；
        
    - BB 继续范围已经被强烈筛选（大量弱牌已弃，只剩带 decent equity / 中强牌）。
        
- turn：
    
    - 出 blank（例如 2♣ 这种不会改变顺子 / 同花结构的小牌）；
        
    - BB 继续保有部分 nuts / 近 nuts，同时很多中段牌 equity 被压缩（要么已经明显落后，要么有足够 redraw）。
        

**solver 结果：**

- 在某些例子里，turn 上 **BB 会以 >60% 频率用 150% pot 再打一枪超池**；
    
- 原因是：
    
    - flop 他已经“投资”让自己留下的是较强的一截范围；
        
    - turn blank 没有改变双方权益对比，他在这条线上的 nuts 比例非常高；
        
    - 再用超池极化，是对 IP flop 超池策略的反制。
        

**重点不是记这些数字，而是记“结构”：**

> **谁的范围在 flop/turn 这条线中被封顶，谁就更容易在 turn 被极化大注针对；  
> 谁能在 turn 保留高比例 nuts，谁就有理由用极化大尺⼨打压对手。**

## 9.6 范围极化：Turn 是从“小注树”切到“极化树”的关键路口

把前几章的内容串一下：

- 第 6 章：你在 flop 选了小注 / 大注 / check，构建了一个“下注树”。
    
- 第 7 章：你在 flop 用 XC / XR / XF 和 check-range 保护，构建了“防守树”。
    
- 第 8 章：你在 flop 通过 raise / donk 做了部分极化。
    

**而第 9 章，就是要告诉你：**

> **turn 是你决定“继续小注滚雪球”，还是“切换到极化大尺⼨ / 超池树”的街。**

### 9.6.1 Turn 的三种典型范围形态

对于任意一条线（比如“flop 小注 cbet → turn 到达”），到 turn 时你的范围大致有三种形状：

1. **仍然合并（merged）：**
    
    - 很多牌都是中等 equity（top pair / overpair / 2nd pair + 各种 draw）；
        
    - 牌面仍然 dynamic，river 还有大量可能。
        
    - 操作：**继续中等尺寸（1/2–2/3）双压，保留多街价值。**
        
2. **开始极化（polar）：**
    
    - 部分牌变成近乎 nuts；
        
    - 大量中段牌 equity 被压缩到“勉强一张牌价值”；
        
    - 操作：**改用大尺⼨（2/3–超池），保留的下注范围是 nuts + 强 bluff。**
        
3. **彻底崩盘（range crushed）：**
    
    - 这张 turn 显著利好对手范围；
        
    - 你的很多 flop bet 组合，到 turn 都偏向“翻车”；
        
    - 操作：**大幅缩减 barrel 频率，转向 check / 放弃。**
        

《全范围 C-Bet 指南 II》里那些“flop 30% → turn/river 70%”的示例，本质就是证明：

> 在很多牌面上，**turn 这一步已经足以让你把范围切成“继续极化压榨 vs 立刻刹车”的两条路**，而不是拖到 river 才决定。

## 9.7 阻挡（blockers）：Turn 上好牌 / 坏牌的分界线

在 flop 上选 bluff 时，阻挡固然重要，但很多人还能凭“直觉”乱抓；  
到了 turn，如果你不认真对待 blockers，**你的 barrel / 抓诈 会非常贵。**

结合 Janda 的逻辑 + 河牌诈唬/抓诈的 PDF（虽然主讲 river，但阻挡逻辑是连贯的），turn 上 blockers 至少有三重作用：

1. **选择继续 barrel 的 bluff**
    
    - 好 bluff：
        
        - 阻挡对手最强的 value 组合（比如阻挡 nut flush / nuts straight）；
            
        - 不过度阻挡对手的 bluff（否则你逼他只剩 value）。
            
    - 坏 bluff：
        
        - 既不阻挡 value，还把对手潜在 bluff 都堵死。
            
2. **选择 XC / XF / XR 的抓诈牌**
    
    - 在 turn 上做 bluffcatch（特别是面对大尺⼨）时：
        
        - 好抓诈牌：
            
            - 自己 showdown value 尚可（top pair / 2nd pair / high card + strong blocker）；
                
            - 阻挡对手一部分 value（比如手上有关键同花牌，减少对方 nut flush 数量）。
                
        - 坏抓诈牌：
            
            - 阻挡的是对手的 bluff（比如自己拿了对方容易用来 bluff 的 A♣ blocker），导致对方理论 bluff 组合减少。
                
3. **选择 turn 极化线中的“纯 bluff”**
    
    - 当你要构建“nuts + pure bluff”的极化 turn 线时：
        
        - 优先选那种 **几乎无摊牌价值、但阻挡结构好** 的牌；
            
        - 例如：
            
            - 阻挡对手 top set / top two 的 card removal；
                
            - 或者双 blocker 某些 nuts 组合。
                

**Turn 阶段，阻挡从“可有可无的细节”变成了“能救命的主逻辑”**：  
你愿意在 flop 上轻微犯错，但在 turn 上，一手错误的 bluff / bluffcatch 通常就是一大口 EV 被扔掉。

## 9.8 临界点：用简单算账把“继续 or 放弃”落地

Turn 上有三个很实用的“临界点”概念，你必须清楚：

### 9.8.1 Bluff 临界点：我需要对手弃多少？

假设：

- pot = P；
    
- 你在 turn bluff 下注 B；
    
- 你 river 几乎不会继续 bluff（把这枪看作“一枪定输赢”）。
    

则你的 bluff break-even 条件是：

> 对手弃牌率 f ≥ B / (P + B)

比如：

- pot=100，下注 70：
    
    - 你需要对手弃 ≥ 70 / (100+70) ≈ 41.2%，bluff 不亏；
        
- pot=100，下注 150（超池）：
    
    - 你需要对手弃 ≥ 150 / 250 = 60%。
        

这就是 turn 极化大注的本质：

- 你给自己设置一个“我需要很高弃牌率”的门槛；
    
- 如果你判断对手在当前结构 + 尺度下确实会弃很多中段牌（权益压缩后的 waist 部分），这枪 bluff 是理论允许甚至推荐的。
    

### 9.8.2 Double barrel 临界点：我这手牌对对手“继续范围”的 equity 是否够？

当你 flop bet → turn 再 bet 时，你要问：

> “在对手 flop 继续 + turn 继续之后的范围上，我这手牌 equity 还有多少？”

简化思路：

- flop 之后对手已经筛走最垃圾的牌；
    
- turn 再继续的，对手多半是：
    
    - 比你弱的中等牌 / draw；
        
    - 部分强 value。
        

你至少希望：

- 在 flop+turn 两枪投入的总筹码，对应的 fold equity + 自己 equity 能支撑这个投入；
    
- 否则，这一枪 turn 很可能已经进入“负 EV”的区域。
    

（这个部分会在第 10 章 double barrel 里具体展开，这里先把临界点概念种下来。）

### 9.8.3 Call 临界点：我需要多少 equity 才能跟？

面对 turn bet B（相对底池 P），你的 call break-even equity 是：

> equity ≥ B / (P + B)

例如：

- pot=100，对手下注 70：
    
    - 你要有 ≥ 41.2% 的胜率，call 才不亏；
        
- pot=100，对手下注 150（1.5x pot）：
    
    - 你要有 ≥ 60% 的胜率，call 才不亏。
        

这在实际决策中，直接对应两个问题：

1. 对手是否真的有那么多 bluff，让你靠中等牌获得这么高的真实胜率？
    
2. 在权益压缩后的 turn 牌面上，你这手牌的 equity 是否已经明显低于这个门槛？
    

**很多“turn hero call 失败”的根本原因，是根本没算过这条线的临界 equity。**

## 9.9 Turn 信息价值最大化的决策模板（Checklist）

以后你在 turn 到达任意一个节点（无论 IP 还是 OOP），可以强行用下面 6 步走一遍：

1. **分类这张 turn：blank / scare card / range shift？**
    
    - blank：对双方 nuts / equity 排名几乎没影响（如无关小牌、重复牌）；
        
    - scare card：
        
        - 补三同花 / 四顺子；
            
        - 或击中明显偏向一方的 high card（比如 UTG vs BB，K 高 turn）。
            
    - range shift：
        
        - 明显偏利被动一方，使其原本 capped 范围突然出现大量 nuts。
            
2. **回顾 flop 的线路：我这条线到 turn 时，谁的范围被封顶？谁极化？**
    
    - range capped：
        
        - 没有太多 nuts，顶端更多是中等价值牌；
            
    - range polar：
        
        - 顶端有大量 nuts / near nuts，底部有空气，中段相对被剥减。
            
3. **判断权益压缩：**
    
    - 这张 turn 有没有把大量“勉强有牌”的组合直接变成垃圾？
        
    - 有没有把某一方的部分组合直接推向 80%+ equity 区间？
        
4. **决定线条形态：继续合并 or 进入极化？**
    
    - 继续合并：
        
        - 仍然有很多中等牌，你计划用 turn 70% / 2/3 继续打，多街榨 value；
            
    - 进入极化：
        
        - 把中段牌更多转入 check / pot control，
            
        - 保留 nuts + bluff 组合在大注 / 超池线里。
            
5. **用阻挡筛选 bluff / bluffcatch：**
    
    - 倾向选择：
        
        - 阻挡 value、多留 bluff 的牌做 bluffcatch；
            
        - 阻挡 value、不挡 bluff的牌做 bluff；
            
    - 避免：
        
        - 阻挡 bluff、给对手留下全部 value 的 combination。
            
6. **算一下临界点：**
    
    - 我这枪 bet / raise 的 size 对应的 “需要弃牌率” 是多少？
        
    - 我 call 对手 size 对应的 “需要胜率” 是多少？
        
    - 在这个范围结构下，对手真的可能给到这些频率吗？
        

**养成习惯：turn 一到，先跑 checklist，再决定点哪颗按钮。**

## 9.10 高频场景：3 个典型 Turn 结构的“模板化”思考

### 场景 1：SRP BTN vs BB，A♠Q♦4♣ | Turn 2♠（blank）

- flop：BTN 30% 小注全范围 cbet，BB 宽 defend；
    
- turn：2♠ 完全 blank，没成顺、没补同花，对应《超池下注指南》里的“范围极化 + BB capped”结构。
    

**结构分析：**

- BTN：
    
    - nuts 区（AQ / set / 两对）仍然完整；
        
    - bluff 区（各类 BDFD / gutshot）中一部分转成强 draw（带 ♠）。
        
- BB：
    
    - 很多弱后门牌 equity 被压缩；
        
    - 顶端多是 AQ / A4 / 44 少量组合，中段一堆 AJo/A9s/中 pocket。
        

**Turn 决策：**

- BTN 适合：
    
    - 把范围极化，用 70%–100% pot 甚至更大尺⼨继续压；
        
    - 价值端：AQ+；
        
    - bluff 端：带 ♠ 的强 overcard / 各类强 draw；
        
- BB 适合：
    
    - 搞清自己在这张牌上的 defend 临界点：
        
        - 中弱 A 高频 XC；
            
        - 边缘 mid pair 开始大量 XF；
            
        - XR 则更多只用于 nuts / 强 draw。
            

### 场景 2：SRP UTG vs BB，J♥T♣4♣ | Turn A♦（scare card+range shift）

- flop：UTG 30% 小注 cbet，BB defend。
    
- turn：A♦ 是 **极强 scare card**：
    
    - 很多 UTG 强 value（AQ/AK/AJ）被补出；
        
    - BB 也有一些 A♣X、两头顺 + A 的 combos。
        

**结构分析：**

- 整体 equity 很可能偏向 UTG（preflop 优势方），
    
- 但 nuts （比如 KQ）可能双方都有；
    
- turn 牌显著改变 both ranges。
    

**Turn 决策：**

- UTG：
    
    - 不能简单照搬 “blank 上继续极化” 模板；
        
    - 要重新评估：
        
        - 自己 nuts 区 vs BB nuts 区的比例；
            
        - 自己中段 AJ/KJ/QJ 这些牌是否仍适合 barreling。
            
- BB：
    
    - 有不少 new strong hand（A+draw），可以作为 XR 候选；
        
    - 同时大量 flop 继续的中对 / 中等 draw 已经被压成废牌，要果断 XF。
        

### 场景 3：SRP BTN vs BB，9♣7♣2♦ | Turn 8♣（range shift 利好 BB）

- flop：BTN 30% 小注，BB 宽 defend；
    
- turn：8♣ 极端利好 BB：
    
    - BB 范围中 65♣ / T6♣ / 各种同花连子 / 顺子 + 同花组合激增；
        
    - BTN 有部分强牌，但整体 nuts 密度明显落后。
        

**Turn 决策：**

- BTN：
    
    - 极不适合继续高频极化 barrel；
        
    - 很多 flop 小注 bluff，turn 直接砍掉；
        
    - 中段 value（overpair / top pair）也需要控制锅。
        
- BB：
    
    - 即便 flop 没 XR，turn 也可以用 XR / lead（半 donk）展示自己的 nuts 优势；
        
    - 对于强 draw + pair 的组合，可以作为 aggression 的核心（XR / lead 大尺⼨）。

## 9.11 常见错误（至少 5 条）

1. **把 turn 当成“再打一枪看看”，不当成整手牌的结构分割点**
    
    - 不分析 blank / scare card / range shift，
        
    - 不看 flop 线路如何封顶 / 极化自己和对手的范围。
        
2. **不尊重权益压缩：**
    
    - 很多 flop 勉强 call 的牌，turn 明显已经在 equity 低谷，仍然“扛一枪”；
        
    - 结果是硬吃两街、到 river 才被迫弃掉，白送 EV。
        
3. **不会在 turn 极化：**
    
    - flop 小注打得挺好，到 turn 依然用类似 1/2 merged 范围乱压；
        
    - 导致三街线条完全不协调，nuts 不能赚足，该 bluff 的牌也没有足够压力。
        
4. **忽视阻挡：**
    
    - turn barrel / XR / 抓诈完全不看 blockers；
        
    - 拿着一堆“阻挡 bluff、不阻挡 value”的烂牌进 bluffcatch 线，或反之。
        
5. **完全不算临界点：**
    
    - 不知道自己这枪 70% pot bluff 需要对手弃多少；
        
    - 不知道面对 150% pot bet 自己 call 需要多少真实胜率；
        
    - 所有决策都停留在“感觉他在偷 / 感觉我牌不差”的层面。
        
6. **把 flop 的 autopilot 延伸到 turn**
    
    - flop 会“自动 cbet”，turn 也“自动 double”；
        
    - 完全不考虑 flop 面/turn 牌对双方范围结构的影响，
        
    - 变成机械化两枪，引来一堆轻松 exploitable 的线。

- 《1.+进阶指南 (2).pdf》
    
    - Janda 对“转牌圈玩法及超池下注”的章节，强调 turn 是“信息增量 × pot 大小”双高的一街，错误成本远高于 flop，并讨论用更强范围在 turn/river 做 check-raise 与 overbet 的 EV 逻辑。
        
- 《Poker_OS_prompt.md》
    
    - 提供本章所沿用的一致结构模板：一句话结论 → 章节目的 → GTO 基线 → 玩家池偏差/临界点 → 决策模板 → 高频场景 → 常见错误 → 迭代日志。
        

【结论】

- 本章建立了 Turn 这一街的核心认识框架：
    
    1. **权益压缩：**大量 flop 还能“看一张”的垃圾牌在 turn 必须被清理，大量原本中段牌的 equity 被推向极端。
        
    2. **范围极化：**turn 是从“小注合并树”切换到“极化大尺⼨树”的关键节点，尤其在 flop 之后一方范围被封顶、另一方保持 nuts 优势时。
        
    3. **阻挡与临界点：**turn 上 blockers 的重要性被放大，而每一个 barrel / call 都必须在必要弃牌率 / 必要胜率这些临界点约束下决策。
