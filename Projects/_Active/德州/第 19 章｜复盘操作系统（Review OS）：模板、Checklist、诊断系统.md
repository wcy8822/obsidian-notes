## 19.1 一句话结论

**强者和普通玩家的差别，不在于谁少犯错，而在于谁有一套“稳定、可重复、能暴露真实 leak 的复盘系统”，让每一手打完的牌都变成下一次升级的筹码。**

---

## 19.2 Review OS 要解决的三个问题

1. **复盘到底复什么？**
    
    - 不是“今天赢了/输了多少”，
        
    - 而是：在关键决策点（尤其是河牌三部曲涉及的 value / bluff / bluffcatch）上，  
        你是否按照既定的 GTO 基线 + exploit OS 在执行。
        
2. **复盘用什么结构？**
    
    - 单手牌：用“Janda 决策逻辑 + 河牌三部曲 + 千算抓诈模板”做模板；
        
    - Session：用 Checklist 检查今天的整体频率、桌子选择、情绪变化；
        
    - 项目长期：用诊断系统识别你在某类 spot 和玩家池 exploit 上的长期模式。
        
3. **复盘结果如何沉淀，而不是看一眼就忘？**
    
    - 千算抓诈锦囊给了一个很明确的习惯：
        
        - 对抓诈/被抓诈手牌打标签，
            
        - 定期集中复盘“同类型节点”；
            
    - 我们要把这个习惯扩展成：
        
        - 在数据库/记牌工具里为关键节点建立“主题仓库”，
            
        - 让复盘从“看几手有印象的牌”升级为“专题诊断”。

## 19.3 复盘的三层结构：手牌 → Session → 项目

你可以把 Review OS 理解成三层：

1. **L1：单手牌复盘（Hand Review）**
    
    - 目标：改善单点决策质量；
        
    - 工具：
        
        - 河牌三部曲提供的典型牌例、计算步骤（pot odds、equity、组合数）
            
        - Janda 的决策逻辑：“下注理由必须明确是 value / bluff / 保护 / 信息（后者通常是错的）”
            
2. **L2：Session 级复盘（Session Review）**
    
    - 目标：看今天整体策略是否偏离自己的 OS；
        
    - 工具：
        
        - 千算锦囊里的“抓诈型 Session 复盘”（集中看河牌大底池）
            
        - 自己的纪律/情绪记录。
            
3. **L3：项目级诊断（Project Review）**
    
    - 目标：识别长期可复用的模式（技术 + exploit + 心理）；
        
    - 工具：
        
        - 数据统计（某类牌面/某类对手的 winrate）；
            
        - “专题仓库”（比如：所有 river hero call / 所有 3bet pot OOP）。
            

下面按这三层展开。

## 19.4 单手牌复盘模板：从“河牌三部曲”开始

《河牌诈唬的进阶指南 I/II》《河牌抓诈进阶指南》本身就是“复盘教程”：

- 选取某手 river 决策，
    
- 把前面街全部展开，
    
- 然后做：
    
    - pot odds 计算；
        
    - 组合数估算；
        
    - 对手类型评估；
        
    - 最后得出“这手是 +EV 还是 -EV”的结论
        

我们把这套逻辑抽象成一个通用模板。

### 19.4.1 通用 Hand Review 模板（Janda × 河牌三部曲）

对任意一手你觉得“关键/怪/纠结”的牌，复盘时写：

1. **场景信息**
    
    - 盲注 / 有效筹码 / 位置 / 牌局类型（现金/MTT）；
        
    - SPR；
        
    - 对手标签（来自上一章玩家池 OS）。
        
2. **行动树简表（preflop → river）**
    
    - 简洁列出每街行动和 size：
        
        - 谁 open / 3bet / call；
            
        - flop/turn/river 的 bet / check / XR / lead 及大小。
            
3. **决策节点定位**
    
    - 哪一街、哪个动作是你要复盘的重点？
        
    - 常见：
        
        - flop cbet or check？
            
        - turn double or give up？
            
        - river value thin or check？
            
        - river bluff / hero call / hero fold？
            
4. **Janda 四问（决策逻辑）**  
    1）这枪下注/跟注的**目的**是什么？
    
    - value？bluff？保护？信息？  
        2）如果是 value：
        
    - 你能被哪些更差牌跟注？频率如何？  
        3）如果是 bluff：
        
    - 对手会弃掉哪些更好牌？频率如何？  
        4）这条线是否有更高 EV 的候选线？
        
    - 比如 check-back / 小码 / 大码 / 另一条街再下注？
        
5. **数学区：pot odds × bluff/value 比例 × 组合数（河牌三部曲）**
    
    - 计算 pot odds：
        
        - call 需要多少胜率？
            
        - bluff 需要多高弃牌率？
            
    - 粗算价值组合 & bluff 组合数：
        
        - 列出 3–4 类典型 value 牌型、对应组合数区间；
            
        - 列出合理 bluff 牌型、组合数区间。
            
    - 判断在当前玩家池倾向下：
        
        - 对手真的会 bluff 到那个比例吗？
            
6. **玩家池 & 对手偏差校正（接入第 18 章）**
    
    - 对手是 [R-underbluff] 还是 [R-overbluff]？
        
    - 是 [R-overcall] 还是 [R-overfold]？
        
    - 在这类人身上，这手决定应该朝 exploit 方向调整吗？
        
7. **结论：这手牌在三条维度上的评估**
    
    - GTO 基线判断：
        
        - 大致是 value / bluff / fold / mix？
            
    - 玩家池 exploit 判断：
        
        - 在这个对手/池子里，哪条线更好？
            
    - 心理执行：
        
        - 决策是否被情绪/结果影响？
            
8. **复盘输出：一句话总结 + 行为指令**
    
    - 一句话：
        
        - “这手本质问题是 turn 没有提前缩窄范围，导致 river 面临过度艰难的 bluffcatch 决策。”
            
    - 行为指令：
        
        - “之后在类似 SPR + 牌面下，对这类对手 turn 优先选择 X 线。”
            

> 要求你在做 Hand Review 时，**必须写完以上 8 点中的至少 5 点**，  
> 否则这次复盘不算完成。

### 19.4.2 “河牌三部曲”专项复盘：三类手牌必须单独仓库

基于《河牌诈唬 I/II》《河牌抓诈指南》的结构，建议你建立三个“专题仓”：

1. **River Value 仓**
    
    - 收集所有：
        
        - 你下注被对手意外跟到底；
            
        - 或你 check-back，摊牌发现自己其实可以 thin value 的手牌。
            
    - 问：
        
        - 我有没有因为“怕被 check-raise 或不被跟注”错失 EV？
            
        - 下一次在同样结构/对手上，这类牌要不要推到更激进一档？
            
2. **River Bluff 仓**
    
    - 收集所有：
        
        - 你做大 bluff 的 hand（成功或失败都算）；
            
    - 问：
        
        - Bluff 三原则是否满足？（几乎无摊牌价值 / 阻挡 value / 不阻挡 fold）
            
        - size 对应的必要 fold 频率在这个对手身上现实吗？
            
        - 这枪 bluff 是否可以提前一街放弃，以减少波动？
            
3. **River Bluffcatch 仓**
    
    - 收集所有：
        
        - 你 hero call / hero fold 的 hand；
            
    - 问：
        
        - 是否用了“抓诈三原则”（阻挡 value / 不阻挡 bluff / 能赢价值底端一部分）
            
        - 玩家池是否整体 underbluff？
            
        - 如果再来一次，你是否要系统性下调抓诈频率？
            

**这三个仓，是 Review OS 的核心数据库。**  
后面 Session / 项目层面的诊断要反复依赖它。

## 19.5 Session 级复盘 Checklist

千算抓诈锦囊很强调“Session 后集中看几手河牌大底池”，而不是随便翻牌谱。我们把这个习惯扩展成一份完整 Session Checklist。

每个 Session 结束后，用 10–20 分钟回答以下问题：

### 19.5.1 技术侧（Strategy）

1. **是否有明显“偏离自己游戏计划”的地方？**
    
    - 例如：
        
        - 明明计划少 4bet，结果被某人 3bet 几次后开始乱 4bet；
            
        - 明明计划减少 hero call，结果被 bluff 一次后开始硬扛。
            
2. **今天最大的三锅：我在 preflop / flop / turn / river 哪一街最不确定？**
    
    - 对每锅写一句：“如果只改一处，我会改哪里？”
        
3. **是否按计划执行 GTO 基线 + exploit 偏移？**
    
    - 有没有在明显 [R-underbluff] 对手身上还去做边缘 hero call？
        
    - 有没有在明显 [R-overcall] 对手身上错失 value？
        

### 19.5.2 心理侧（Mental）

1. **今天有没有出现 B-Game / C-Game？**
    
    - 若有：
        
        - 触发点是什么？（连续 coolers / bad beat / 社交干扰）
            
        - 具体表现是什么？（冲动 4bet、无计划 bluff、连续 call 不该 call 的牌）
            
2. **停牌机制是被动触发还是主动触发？**
    
    - 是因为“资金达到 stop-loss”才停，
        
    - 还是在意识到进入 C-Game 时主动停？
        

### 19.5.3 结构侧（Table & Volume）

1. **今天桌子质量如何？**
    
    - 是否有刻意寻找鱼桌 / 避开明显 tough reg 桌？
        
    - 有没有因为 tilt 留在差桌上硬打？
        
2. **手数 & 精力分配是否合理？**
    
    - 越到后面，决策质量是提高/持平/下降？
        
    - 若下降，后半场的数据是否明显比前半场差？
        

### 19.5.4 Session 小结（必须写下来）

用“三行总结”结束每次 Session：

1. 今天最值得保留的一个决策模式（做对的部分）；
    
2. 今天最需要修正的一个模式（典型 leak）；
    
3. 明天 Session 想刻意练的一个点（一个就够）。
    

> 要求：每天 Session 复盘不要超过 30 分钟，  
> 但要坚持写完这份 Checklist。

## 19.6 项目级诊断系统：从“感觉”到“系统性 leak 地图”

当你有了上面三个“河牌仓 + Session 小结”之后，就可以做更高级的诊断。

### 19.6.1 技术诊断维度

每隔一段固定周期（比如每月/每 5w 手），从数据库 + 仓库中提这些问题：

1. **河牌三部曲诊断**
    
    - River Value 仓：
        
        - 你出现“本该 value 却 check-back”的频率有多高？
            
    - River Bluff 仓：
        
        - 你的 bluff 是否集中在结构合理的牌面？
            
    - River Bluffcatch 仓：
        
        - hero call 赢率 vs 理论所需胜率差距有多大？
            
2. **玩家池 exploit 执行度诊断**
    
    - 对 [R-underbluff] 类型：
        
        - 你是否已经系统性减少了 bluffcatch？
            
    - 对 [R-overcall] 类型：
        
        - 是否明显增加 thin value？
            
3. **决策逻辑诊断（Janda 四问）**
    
    - 从复盘记录中统计：
        
        - 有多少次下注/跟注的“理由”写不出来，  
            证明当时其实是“瞎按”？
            

### 19.6.2 心理 & 波动诊断

结合第 17 章波动与心理：

1. **在 downswing 期，你的决策质量是否显著下降？**
    
    - 对比 upswing / 平稳期：
        
        - 是否 downswing 时你的 hero call / tilt bluff 明显增多？
            
2. **Session 停牌触发方式统计**
    
    - 一周内：
        
        - 有多少次是因为达 stop-loss 停？
            
        - 有多少次是因为自觉进入 C-Game 停？
            
    - 长期目标：
        
        - 提升“主动停”比例，降低“被动爆仓式停”的比例。

## 19.7 用 Janda 决策逻辑，给自己一套标准“复盘问卷”

为了避免复盘时“看着牌谱发呆”，可以直接把 Janda 的决策逻辑做成一套问卷，对每个关键节点都问：

1. **我这里的下注/跟注，是 value 还是 bluff？**
    
    - 如果你写出“为了信息”/“因为他很烦”，
        
    - 这手就直接判定为“逻辑不合格”。
        
2. **如果是 value：**
    
    - 我希望更差的哪些牌跟注？
        
    - 对手现实会跟多少？（玩家池角度）
        
3. **如果是 bluff：**
    
    - 对手会弃掉哪些更好牌？
        
    - 这个节点他 bluff/underbluff 的历史印象如何？
        
4. **范围视角：**
    
    - 在我的整条 range 里，
        
    - 这手牌处于哪一层？（顶层 / 中上 / 中间 / 底端）
        
    - 我是否已经在这条线弃掉了比它更强的牌？
        
5. **数学兜底：**
    
    - 这枪 call/ bluff 需要多少胜率 / fold 频率？
        
    - 在这个对手身上是否 realistic？
        

你可以强制要求自己：

- 每天至少用这套问卷完整复盘 3–5 手，
    
- 把“讲故事的复盘”变成“答卷式复盘”。

## 19.8 把“千算抓诈锦囊式复盘”扩展到全局

抓诈锦囊的一个核心思想是：

> 不要零散看牌，要 **集中看某一类节点**，  
> 比如“这周只看河牌 hero call”。

我们扩展为“专题复盘 OS”：

1. **选一个主题**
    
    - 示例：
        
        - “BTN vs BB 翻牌 cbet 结构”；
            
        - “3bet pot OOP 防守”；
            
        - “对抗激进玩家的 XR 应对”；
            
        - “河牌 thin value 决策”。
            
2. **从数据库 / 仓库中抽样 30–50 手相关牌谱**
    
    - 只看和主题高度相关的手牌；
        
    - 不要混杂其他节点。
        
3. **对每手应用同一套模板**
    
    - 比如：
        
        - 对 thin value 仓，用河牌 value 章节里的三问：
            
            - 真的有更差牌跟？
                
            - 有没有被过多更好牌跟？
                
            - 是否比 check EV 高？
                
4. **从样本中抽象出自己的“倾向”**
    
    - 例如：
        
        - “我几乎不会在 IP 干面上 check-back 强牌”；
            
        - “我在 3BP OOP 里 overfold turn”；
            
        - “我对激进玩家的 XR 过于被动，只会 call，不会 3bet。”
            
5. **给出一条具体行为规则**
    
    - 每个主题复盘最后，
        
    - 总结成 1–3 条“下次必须执行的简单规则”，
        
    - 然后写进你的 Poker OS 行动路线。
        

**这样做的结果是：**  
复盘不再是“看几手印象深的牌”，而是每个周期系统地修一个模块。

## 19.9 常见错误（至少 8 条）

1. **把复盘当“看输赢过程”，不当“诊断系统”**
    
    - 只看那些赢巨锅/输巨锅的牌，
        
    - 完全不看自己频繁出现的中等底池决策模式。
        
2. **复盘只看结果，不看决策质量**
    
    - 赢了就觉得打得好，输了就觉得打得差，
        
    - 完全无视数学 & 结构。
        
3. **不写东西，只在脑子里“过一遍”**
    
    - 没有文字输出，就没有可复用的模式沉淀；
        
    - 每次复盘都像“从零开始思考”。
        
4. **复盘范围过大，缺乏主题**
    
    - 一次打开上百手牌乱看，
        
    - 最后印象就是“好乱”，
        
    - 没有任何明确可执行的改动点。
        
5. **复盘只看技术，不看心理与桌子选择**
    
    - 忽略了很多 EV 损失其实来自：
        
        - tilt 决策、
            
        - 坏桌子上死扛、
            
        - 过度疲劳打牌。
            
6. **用复盘来“证明自己是对的”**
    
    - 只找支持自己 original 决策的论据，
        
    - 避开所有相反证据，
        
    - 变成自我安慰，而不是自我修正。
        
7. **复盘没有频率，也没有标准流程**
    
    - 想起就看两手，
        
    - 想不起来就几天不看，
        
    - 缺乏“每天 / 每周 / 每月”的固定节奏。
        
8. **复盘完从不写“下一步行动”**
    
    - 看完“哦，原来如此”，
        
    - 下一次实战照旧，
        
    - 导致同一个 leak 一年都不变。
