# 第 13 章｜河牌诈唬（Bluff）系统：阻挡、未阻挡、摊牌价值、EV

## 13.1 一句话结论

**河牌诈唬不是“我感觉他会弃”，而是：在理解双方范围后，用几乎没有摊牌价值、同时“阻挡对手跟注牌、不要阻挡对手弃牌”的组合，在合适的 size 下做一枪高 EV 但高波动的非稳定胜率决策。**

---

## 13.2 本章要解决的三个问题

1. **我用什么牌去 bluff？**
    
    - 《河牌诈唬进阶指南 II》明确给出 “诈唬三原则”：
        
        - 几乎没有摊牌价值；
            
        - 阻挡对手的跟注范围；
            
        - 不阻挡对手的弃牌范围。
            
    - 我们要把这三条变成可以套用的“选牌规则”。
        
2. **我在什么结构 / 线路下 bluff？**
    
    - 不是任何 river 都适合 bluff：
        
        - 有些牌面对手范围极强，你 bluff 频率要降到接近 0；
            
        - 有些牌面对手 range capped，你反而应该有系统的 bluff 频率。
            
3. **我用什么 size、需要什么弃牌率，EV 才合理？**
    
    - 河牌 bluff 是“高波动、高杠杆”的决策，
        
    - 你必须知道：
        
        - 下注 size 对需要弃牌率的影响；
            
        - 在你认知的玩家池里，这个弃牌率现实不现实；
            
        - 这枪 bluff 属于“长期可复制的 +EV”，还是“纯赌对方这一次刚好会弃”。

## 13.3 理论基线：河牌 bluff = 典型“非稳定胜率”动作

### 13.3.1 Janda & 《德州扑克的波动》：高 EV ≠ 低波动

在 Janda 的框架 + 《德州扑克的波动》中有一个共识：

- 扑克中的决策可以大致分成：
    
    1. **稳定胜率（low variance 高胜率）：**
        
        - 比如拿 nuts 在小底池里 value bet；
            
        - 每一手赢少一点，但几乎不出事。
            
    2. **非稳定胜率（high variance 高胜率）：**
        
        - 比如 river 极化 bluff / hero call / 巨额 thin value；
            
        - 每次 EV 可能是正的，但单手波动非常大。
            

河牌 bluff，尤其是大尺⼨ bluff，毫无疑问属于后者：

- 做得好：长期大幅提升你在高水平池子里的 winrate；
    
- 做不好：短期样本下看起来像“自杀行为”，心态极易崩。
    

**你要接受的现实是：**

> 河牌 bluff 天生就“输多赢少”（从次数上看），  
> 但每次赢赢一大锅，每次输输一大锅，  
> 长期靠数学 + 频率 + 结构赚那一点边际 EV。

所以本章的目标不是教你“每次都 bluff 成功”，而是：

- 让你知道 **什么时候必须 bluff 一部分**，
    
- 以及 **选哪些组合来 bluff，才不会把自己的 EV 烧光。**

## 13.4 《河牌诈唬进阶指南》的核心：诈唬三原则

原文把“河牌诈唬三原则”写得非常干脆（我们抽象为 OS 版）：

> 选择河牌 bluff 牌时，要优先满足三个条件：  
> 1）几乎没有摊牌价值；  
> 2）阻挡对手的跟注范围；  
> 3）不过度阻挡对手的弃牌范围。

### 13.4.1 原则一：几乎没有摊牌价值

原文直接举了一个非常典型的心理场景：

- 你在河牌犹豫要不要 bluff，
    
- 最后怂了，选择 check，结果摊牌时你靠一张高牌赢了底池。
    

**结论：**

- 用这种“其实有 decent 摊牌价值”的牌去 bluff，
    
    - 属于高风险低回报；
        
    - 因为你把原本可以“免费赢一部分底池”的手牌改成了“非赢即输的大注”。
        

所以在实战中，**优先拿下面几类牌做河牌 bluff：**

1. 所有 draw 想到 river 仍然完全 miss 的牌（无对、无靠谱 kicker）；
    
2. turn 半诈唬的强 draw（如 NFD+OESD）在 river 完全砖掉；
    
3. 一些在前面街只是被动跟注的弱 pair / 高牌，在当前牌面上几乎不可能赢下摊牌。
    

**反例：**

- 各种 TP 弱 kicker / 2nd pair / 小 overpair，
    
    - 在很多牌面上对手会 check-back 一堆更差牌，
        
    - 拿这些牌去 bluff，往往是“把能 show down 赢的 EV 全扔掉”。
        

### 13.4.2 原则二：阻挡对手的跟注范围（block calling range）

经典例子（简化版）：

- SB 3bet，BTN call；
    
- flop：6♠6♥5♣，SB 小注 cbet，BTN call；
    
- turn：2♥，SB 70% double，BTN call；
    
- river：J♦，SB 考虑 all-in bluff。
    

在原文中问你：**T9s vs AKs，谁更适合 bluff？**

- T9s 阻挡了 BTN 可能用来抓诈的 TT / 99；
    
- AK 几乎不阻挡这些抓诈牌。
    

同时：

- AK 还有一定摊牌价值（打赢一些 A5/A4 弱对），
    
- T9 完全没摊牌价值。
    

**所以：T9 是 river bluff 的更优候选。**

你要内化的东西是：

> 河牌 bluff 要 **减小对手“能 call 的强牌”数量**，  
> 你的手牌越多地阻挡了对方抓诈组合，  
> 对方在组合意义上就越难找到足够的 call。

### 13.4.3 原则三：不阻挡对手的弃牌范围（unblock folding range）

原文之后继续在同一条线路上追问：

> “OK，T9 里哪种花色最不适合 bluff？”

答案是：**阻挡了大量弃牌组合的那种花色**，  
比如牌面上两颗红心的结构里，T♥9♥ 会挡掉大量转牌跟注、河牌弃牌的红心听牌组。

换句话说：

- 如果你的 bluff 牌刚好挡住了对手一大堆会在河牌弃掉的 draw / 弱对，
    
- 你的 bluff 成功率会被大幅拉低。
    

**三原则合在一起：**

1. 这手牌本身秀下去没啥希望（低摊牌价值）；
    
2. 它能减少对手继续跟注的好牌（阻挡 call）；
    
3. 它不减少对手弃掉的垃圾牌（不阻挡 fold）。
    

符合的，才是**河牌 bluff 的“黄金组合”**。

## 13.5 牌型选择：从“最差牌”开始筛

结合三原则，我们可以给你一个实用流程：

### 13.5.1 第一步：清理掉“有摊牌价值”的牌

- 在当前线路下，**先把“有 decent 摊牌价值”的牌全部从 bluff 候选里剔除**：
    
    - 各种 top pair / 2nd pair / overpair；
        
    - 各种偶尔能赢对手“空牌”的高牌；
        
- 这些牌更适合：
    
    - check-back / check-call，
        
    - 或者在极少数结构中做 thin value（上一章的内容）。
        

### 13.5.2 第二步：从 miss draw 中挑“最好的一批”

- 典型 bluff 仓库：
    
    1. 完全 miss 的同花听牌 / 顺子听牌；
        
    2. turn 半诈唬未成的 combo draw；
        
    3. 一些 flop 以来一路跟到 river 的 high-card + backdoor draw 垃圾牌。
        
- 在这些 miss draw 中：
    
    - 用三原则筛一遍：
        
        - 优先选：
            
            - 阻挡对手抓诈牌（call range），
                
            - 不阻挡对手放弃牌（fold range）。
                
        - 淘汰：
            
            - 阻挡了大量 draw + 小对（这些是对手现实弃牌的主力）。
                

### 13.5.3 第三步：必要时才考虑“把部分中 value 转成 bluff”

在很多 solver 解里，你会看到：

- 某些中等牌力（比如 river 的最弱 top pair / 2nd pair）
    
    - 被混入一部分 bluff 频率，
        
    - 用来弥补 bluff 组合不够、对手过多 overfold 时的 exploit。
        

在玩家池现实中，这一块是**你最后才需要碰的高级选项**，理由很简单：

- 微级别玩家面对大码河牌，整体 overfold；
    
- 你根本不用冒险去“强行把中 value 转成 bluff”，
    
- 只要用 miss draw 仓库 + 少量垃圾牌，
    
- 已经足够覆盖你应该有的 bluff 频率。

## 13.6 结构与线路：不是每条线都值得 bluff

### 13.6.1 自己代表什么？对手代表什么？

在河牌之前，你已经选定了一条线（cbet→double→? / XC→XC→? / XR→? 等）。

- 河牌你要 bluff 时，必须先回答两句：
    
    1. **在这条线里，我的 nuts 比他多还是少？**
        
    2. **在这条线里，他的中段 bluffcatch 是否过多？**
        

如果答案是：

- 我 nuts 明显多，他 bluffcatch 一堆 → 适合建 bluff 线；
    
- 他 nuts 明显多，我被 capped → 这条线上的 bluff 频率就该非常低。
    

### 13.6.2 典型适合 bluff 的结构

- 你保留 nuts 优势，对手在路上从未 XR：
    
    - 例如 IP 在 AKQ/高协面一路压制，
        
    - OOP 只 call，从未在 turn/river 表现极化；
        
- 河牌对 nuts 结构影响不大（blank / 轻微 scare）：
    
    - 不会突然把大部分 nuts 转移到对手那边；
        
- 对手前几街表现“防守型”：
    
    - flop/turn 都是“不得不 call”的 defend，
        
    - 你 river 极化 bluff 是在惩罚他过多扛前两街。
        

### 13.6.3 典型不适合 bluff 的结构

- 河牌明显利好对手范围（range shift）：
    
    - 补低端顺 / 三同花 / 现场常见的一堆 scary card；
        
- 对手在 turn 已经展现强烈极化（XR / 大码 lead）：
    
    - 他在河牌出现 nuts 的比例非常高；
        
- 在你这条线中，你的 nuts 几乎比不过他：
    
    - 比如你是被动 call 方，一路没 XR，
        
    - 现在突然在大 scary river 上 bluff lead，
        
    - 在组合意义上非常难自圆其说。
        

**千算攻击篇给的结论其实很实在：**

> 绝大多数业余玩家，河牌 bluff 的问题不是“太少”，  
> 而是选错了结构、选错了对手、选错了牌。

## 13.7 数学区：河牌 bluff 的 EV 与 size

### 13.7.1 单枪纯 bluff 的最简公式

设：

- pot = P；
    
- 你在 river bluff，下注 B；
    
- 对手弃牌率 = f；
    
- 被跟注时，你的牌 100% 输（典型纯 bluff）。
    

则这枪 bluff 的 EV 为：

[  
EV = f\cdot P ;-; (1-f)\cdot B  
]

盈亏平衡条件（EV = 0）：

[  
fP = (1-f)B  
\Rightarrow f(P+B) = B  
\Rightarrow f = \frac{B}{P+B}  
]

几个常用 size 下的 **最小需要弃牌率**：

- **1/2 pot（B=0.5P）**
    
    - f = 0.5 / 1.5 ≈ 33.3%
        
- **2/3 pot（B≈0.67P）**
    
    - f ≈ 0.67 / 1.67 ≈ 40%
        
- **pot（B=P）**
    
    - f = 1 / 2 = 50%
        
- **1.5x pot（B=1.5P）**
    
    - f = 1.5 / 2.5 = 60%
        

**你必须挺直腰板承认：**

- 你在 micro 以 1.5x pot bluff，要对手弃 60% 才不亏，
    
- 如果对手是一个“英雄跟注型”，你这枪很容易变成—EV。
    

### 13.7.2 有一点 equity / 阻挡时的修正

在河牌，绝大多数 bluff 组合 equity ≈ 0（要么赢要么输）。

但在少数结构中：

- 你的 bluff 可能打赢对手一些更差 bluff（冷碰 bluff）；
    
- 或者打平某些 chop。
    

简单理解就行：

- 如果你 bluff 的牌仍然有**少量摊牌价值**，
    
    - 你实际所需弃牌率会略低于公式值；
        
- 但请注意，这和“第一原则：几乎没有摊牌价值”并不冲突：
    
    - 这里讲的是“剩余一点 residual equity”，
        
    - 而不是那种可以稳定赢一小撮底池的 top pair/2nd pair。
        

### 13.7.3 频率视角：bluff:value 与 MDF（概念性）

在 solver 视角，bluff:value 比例 + 对手的最低防守频率（MDF）存在平衡关系：

- size 越大：
    
    - 合理 bluff 组合比例越低；
        
    - 对手的 MDF 越低（可以多 fold）。
        

但在微级别实际：

- 大注下去，对手 MDF 往往 **低于 GTO 要求**（过度弃牌），
    
- 所以你理论上可以用比 solver 更多的 bluff，
    
- 或者用比 solver 更 value-heavy 的范围（看对手倾向）。
    

本章不要求你记具体比例，  
只要记住两个方向：

- 对 **怕钱玩家**：
    
    - 你可以 value-heavy，
        
    - 不必追求 GTO bluff 频率。
        
- 对 **英雄跟注玩家**：
    
    - 你必须大幅减少大尺寸 bluff，
        
    - 同样结构，用更多 thin value 替代 bluff。

## 13.8 玩家池 exploit：千算攻击篇的河牌 bluff 规则（抽象版）

虽然原 PDF 是图片为主，我们抓不到逐字，但核心思想很好概括：

### 13.8.1 对“只打价值”的鱼：河牌 bluff 频率压到极低

- 特征：
    
    - 自己很少 bluff；
        
    - 面对大注基本只用 nuts / 很强牌跟注；
        
    - 只在“有牌”就 bet，没牌就 check。
        

对这类人：

- 你不需要“平衡”；
    
- 在绝大多数结构上，**干脆不要河牌 bluff**：
    
    - 你的钱主要来自 value bet；
        
    - 偶尔的 bluff 也选最标准的 combo（严格遵守三原则），
        
    - 不需要追求任何精致频率。
        

### 13.8.2 对“激进 reg / over-cbet / over-bluff 型”：河牌 bluff 适度提高

- 特征：
    
    - 喜欢多 cbet、多 barrel、多 XR；
        
    - 也会在 river 上适度 bluff。
        

对这类人：

- 你可以更接近 GTO frequency：
    
    - 在 nuts 优势结构中，给出一定比例 river bluff；
        
    - 选牌严格按三原则；
        
    - 尺寸根据牌面结构与自己整套 overbet/正常 size 策略来定。
        

### 13.8.3 对“英雄跟注玩家”：河牌 bluff 频率明显降低甚至 0

- 特征：
    
    - 面对大注，习惯性“我不信”，
        
    - 拿一堆 bluffcatch 扛到底。
        

对这类人：

- 你可以几乎不 bluff（尤其是大尺⼨）：
    
    - 把所有潜在 bluff 组合的一部分改成 check-fold / check；
        
    - 把另一部分转成 flop/turn 早期放弃。
        

千算体系的建议很简单粗暴：

> **面对过度 hero call 的人：  
> 河牌大注 = value only。**

你不用和 solver 辩论，  
直接围绕对手钱袋子设计策略。

## 13.9 河牌 bluff 决策 Checklist

以后到任何一个河牌节点，你有 bluff 冲动时，强制自己跑完这 7 步：

1. **这手牌有摊牌价值吗？**
    
    - 如果“有一定摊牌价值”：默认不 bluff，考虑 check-call / check-fold；
        
    - 如果“几乎没摊牌价值”：可以进入候选。
        
2. **在当前线路下，我 represent 的是极化范围吗？**
    
    - 如果你在整个手牌中都表现得偏合并，
        
    - 突然在河牌打出一次极化大注，
        
    - 在聪明对手眼里会非常可疑。
        
3. **对手在这条线中被 capped 吗？谁 nuts 多？**
    
    - 如果 nuts 明显集中在你这一边，对手多是 bluffcatch：
        
        - 结构适合 bluff；
            
    - 如果 nuts 集中在对手那边，你被 capped：
        
        - 结构极度不适合 bluff。
            
4. **我的这张牌是否阻挡对手跟注范围？**
    
    - 有没有阻挡他常见抓诈牌（比如 top pair / 两对 / nut blocker）？
        
    - 如果没有阻挡，可能只是“裸 bluff”，成功率偏低。
        
5. **我的这张牌是否避免阻挡对手弃牌范围？**
    
    - 别拿阻挡大量 miss draw / 小对 / 弱 pair 的牌 bluff；
        
    - 那会直接削弱你的 fold equity。
        
6. **对手是哪一类型（怕钱 / 正常 / 英雄）？**
    
    - 怕钱：可以适度增加 bluff 频率（尤其是中小码 bluff）；
        
    - 英雄：大注 bluff 几乎砍光。
        
7. **size 对应的必要弃牌率现实吗？**
    
    - 1/2 pot 需要弃约 33%；
        
    - pot 需要弃 50%；
        
    - 1.5x pot 需要弃 60%。
        
    - 在你对这个人和这个牌面的判断里，他真的会弃到这个频率吗？
        

**Checklist 跑完如果答案仍然偏“YES”，那这枪 bluff 至少是“有理有据”，不是情绪拍脑袋。**

## 13.10 常见错误（至少 7 条）

1. **用有明显摊牌价值的牌 bluff**
    
    - 各种 top pair / 2nd pair / 小 overpair 被拿去 bluff，
        
    - check-down 本来可以赢，却变成“非赢即输的高波动决策”。
        
2. **不看对手范围，盲选 miss draw bluff**
    
    - 只要是 miss draw 就 bluff，
        
    - 完全不考虑是否阻挡对手抓诈牌 / 弃牌牌，
        
    - 导致 bluff 成功率和理论期望相去甚远。
        
3. **在对手 nuts 明显更多的河牌结构上硬 bluff**
    
    - 典型：低端顺 / 三同花 / paired board 极利 OOP，
        
    - IP 还在那儿装代表 nuts 极化 bluff，
        
    - 长期就是对着对手 nuts 烧钱。
        
4. **在对手极度怕钱的池子里追求“平衡”**
    
    - 为了“看起来像 reg”，在明显 overfold 池子里照抄 solver bluff 频率，
        
    - 最终干的事是：
        
        - 把本应靠 value 轻松兑现的 EV 用 bluff 废掉一大块。
            
5. **在对手英雄跟注的池子里继续大码 bluff**
    
    - 对 “永远不信你”的玩家仍用 overbet / pot bluff，
        
    - 一遍遍用自己的 stack 给对方做心理按摩，
        
    - 然后得出结论“河牌 bluff 不赚钱”，完全是方向搞反。
        
6. **不算 size 的必要弃牌率**
    
    - 完全不知道 1.5x pot 需要对手弃 60%，
        
    - 靠“感觉他要弃牌了”决策，
        
    - 这在高波动 spot 中几乎必然长期—EV。
        
7. **线不连贯：前面像 value 到 river 像 bluff，或反之**
    
    - flop/turn 用 value-line，river 突然代表一堆 miss draw 来 bluff；
        
    - 或 flop/turn 一直 over-bluff，river 又装成 nuts-only；
        
    - 线条极度可读，给对手 exploit 空间。
