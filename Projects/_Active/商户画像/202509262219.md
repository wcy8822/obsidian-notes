## 三段式逻辑支撑

### 1) 关键逻辑链条（主要推演与决策逻辑）

- 先**看基本面**：用当日快照从重叠和品牌两个维度刻画供给结构（规模=Sum；质量=P50±P25/P75；执行=fee_gap P50）。
    
- 再**找机会**：
    
    - 重叠：同城×同品牌对照，算 `ΔP50(无重叠−有重叠)`；
        
    - 品牌：同城相邻档对照，算 `ΔP50(上一级−当前档)`。
        
- **量化收益**（两条）：`uplift_benchmark=ΔP50×GMV` 与 `uplift_contract=fee_gap P50×GMV×回收率(30/50/70%)`；保守口径取二者**较小**。
    
- **判定显著性**：红绿灯阈值（重叠≥30bp、品牌≥50bp，且 N≥30）；不显著再做**交叉控制**（在重叠中已控品牌；在品牌中分 o=0/o=1）。
    
- **输出与承诺**：合并机会为 Top 清单（全国→Top 省区→Top5 城市），附动作占位与责任人。
    

### 2) 方法论与框架（可迁移的思路与模型）

- **四步法**：结构体检 → 横截面对照（ΔP50）→ 机会排序 → 保守 uplift。
    
- **统计口径**：规模看 **Sum**；质量看 **分位数**（P50±P25/P75）；对照看 **ΔP50**；稳定性看 **IQR** 与 **样本量 N**。
    
- **阈值科学定位**：三把尺子校准——**稳定性**（bootstrap 95%CI 宽度）、**经济性**（uplift≥U_min）、**一致性**（TopK Jaccard）。
    
- **地域选择**：全国总览 → Top 省（规模或机会）→ Top5 城。
    
- **站点分层 v0.1**：A 成熟 / B 潜力上拉 / C 治理机会（以城市内百分位 + Δ/fee_gap 判定）。
    

### 3) 数据与事实（关键数字、定义、取舍点）

- **当日快照**，无回溯；只做横截面。
    
- **指标口径**：
    
    - 订单/GMV：Sum；
        
    - 实际费率、合同费率、单利、订单密度：P50±P25/P75（合同费率可仅 P50）；
        
    - fee_gap=合同−实际（看 P50，gap>0 才计回收）。
        
- **阈值默认**：N_min=30；重叠 δ=30bp；品牌 δ=50bp；覆盖率<80% 或 99>20% → 观察；回收率 30/50/70%。
    
- **收益口径**：`uplift_conservative=min(uplift_benchmark, uplift_contract@50%)`。
    
- **数据质量**：未知值（99）不入主结论，单列覆盖率。
    
- **公平对照**：重叠必须在**同城×同品牌**内比，避免品牌/城市结构混淆与辛普森悖论。

## 深度展开（隐含假设 / 风险与盲区 / 体系化升级）

**隐含假设**

- 标签当日快照在短期内相对稳定；
    
- 合同费率代表常态基线，fee_gap 中的可回收部分可在 4–8 周内落地；
    
- `ΔP50` 能在同城同品牌内代表“典型差异”。
    

**潜在风险与盲区**

- **样本稀疏**：小城或小品牌 N 不足，Δ 不稳；
    
- **覆盖不齐**：未知=99 较高时结论偏差；
    
- **执行刚性**：fee_gap 中结构性让利不可回收；
    
- **横截面非因果**：仅用于优先级排序，不代表因果提升；
    
- **权重劫持**：大城/大盘主导全国口径，需并行等权对照。
    

**体系化升级**

- **阈值面板化**：N_min / δ / 回收率三拨杆 + 实时 TopK 稳定度；
    
- **站点分层产品化**：A/B/C 层与动作库映射（谈判/权益/产品/合同执行）；
    
- **复盘闭环**：自动追踪“合同回收兑现率”“结构对标贴近度”，定期调参；
    
- **数据治理看板**：覆盖率与 99 占比红线驱动标签治理优先级。
