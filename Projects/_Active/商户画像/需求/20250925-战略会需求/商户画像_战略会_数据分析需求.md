# 北极星目标（一句话）

用**两条视角（重叠 / 品牌）**在**当日快照**下读清**供给基本面**，找出**显著机会位**并量化**保守 uplift**，形成**全国 → Top 省区 → Top5 城市**的决策清单与会后承诺页。

>会议故事线
> 先看**基本面** → 再找**机会** → 若不显著再做**交叉控制** → 生成**Top 清单**与**承诺**。



## 1. 总策略与流程（先简单，后交叉）

**总策略**：

1. **基本面**（两条线独立看）：

   * 重叠：o=0 / o=1 的渗透、规模与质量中位（费率/单利/订单密度），并报合同费率与 fee\_gap。
   * 品牌：KA / CKA / 小散的渗透、规模与质量中位，并报合同费率与 fee\_gap。
2. **机会**（两条腿并行）：

   * 重叠腿：同城×同品牌内看 **ΔP50(费率)=无重叠−有重叠**（单利/订单密度作旁证），做**红绿灯**与**uplift**。
   * 品牌腿：同城内看 **ΔP50(费率)** 的相邻档差（KA−CKA、CKA−小散），做**红绿灯**与**uplift**。
3. **交叉控制（按需）**：若两条腿结论为黄/红但业务希望深挖，则：

   * 重叠腿：已天然控制品牌；必要时合并至母品牌/品牌族后再判。
   * 品牌腿：分别在 **overlap=0 / overlap=1** 下单独判，去除竞争扰动。
4. **汇总与排序**：合并两条腿的机会，按**uplift\_conservative** 从高到低排序；每行附**建议动作占位**（谈判/权益/产品/合同执行）。
5. **地域层级**：只看 **全国 → Top 省区 → Top5 城市**；不全扫 300+ 城。

## 2. 报告骨架（DS 交付结构，不限定图形）

### 2.1 基本面（两张总表）

* **重叠视角**：o=0 / o=1 在全国 → Top 省区 → Top5 城市的：

  * 规模：站点数、订单 Sum、GMV Sum；
  * 质量：实际费率 **P50 ± P25/P75**、单利 **P50 ± P25/P75**、订单密度 **P50 ± P25/P75**；
  * 执行：合同费率 **P50**、**fee\_gap P50=合同−实际**；
  * 质量：覆盖率%、未知=99%、样本量 N。
* **品牌视角**：KA / CKA / 小散，同上口径。

### 2.2 机会（两条腿）

* **重叠腿**：输出“城市×品牌”粒度的：`ΔP50(费率) / ΔP50(单利) / ΔP50(订单密度)`、N、GMV、`uplift_benchmark`、`uplift_contract_p30/50/70`、`uplift_conservative`、**红绿灯**。
* **品牌腿**：输出“城市×品牌×档位”粒度的：相邻档 `ΔP50(费率)`、N、GMV、两类 uplift、**红绿灯**。

### 2.3 管理层摘要（全国 → Top 省区 → Top5 城市）

* 3–5 条**基本面要点** + 3–5 条**机会要点**（含规模与保守 uplift 区间）。
* **Top 机会清单**：按 `uplift_conservative` 排序，带建议动作占位与责任人占位。

## 3. 显著性阈值（红绿灯规则）

* **重叠腿**：

  * 绿：`N≥30` 且 `ΔP50(费率) ≥ 30bp`，且 `P25(无重叠) ≥ P75(有重叠)`；
  * 黄：`N≥10` 且 `ΔP50 ∈ [10,30)bp`；
  * 红：其他（仅观察）。
* **品牌腿**：

  * 绿：`N≥30` 且 `ΔP50(费率) ≥ 50bp`，且 `P25(上一级) ≥ P75(当前档)`；
  * 黄：`N≥10` 且 `ΔP50 ∈ [20,50)bp`；
  * 红：其他。

> 阈值做成参数：重叠 20/30/40bp、品牌 40/50/60bp；会场可切档。

---

## 4. 地域层级选取

* **全国**：总览与结论口径。
* **Top 省区**（二选一）：

  * 规模导向：按 GMV 占比 Top（如 Top8）。
  * 机会导向：按 `uplift_conservative` Top（保证 N 与覆盖率达阈）。
* **Top5 城市**：沿用省区选择规则在重点省内选 Top5。

## 5. 站点价值分层（v0.1 当日快照版）

* **A 层（成熟/金牌）**：同城×同品牌内，`rate_actual ≥ P75` 且 `overlap=0`，`fee_gap P50 ≤ 20bp`，`order_density ≥ P50`。
* **B 层（潜力/上拉位）**：`rate_actual ∈ [P50, P75)`，且 `ΔP50(对标) ≥ 30bp` 或 `fee_gap P50 ≥ 30bp`。
* **C 层（机会/治理位）**：`rate_actual < P50` 且 `ΔP50 或 fee_gap` 显著（≥50bp），并满足最小规模阈值（GMV 或订单）。

> 阈值参数化：20/30/50bp 可切换；仅用于**派单优先级**，不做因果宣称。

## 6. 指标 × 口径 × 价值说明（总表）

> 规则：**规模看 Sum；质量看 P50±分位；对比看 ΔP50；执行回收看 fee\_gap × 回收率**。所有结论旁必须带 **N** 和覆盖率。

| 字段/派生                       | 粒度          | 计算口径                         | 价值/用途        | 定义注释             |
| --------------------------- | ----------- | ---------------------------- | ------------ | ---------------- |
| orders\_sum（订单）             | 全国/省/城/品牌/站 | **Sum**                      | 规模、排序、阈值     | 自然日 T+1 完整日汇总    |
| gmv\_sum（GMV）               | 同上          | **Sum**                      | 规模、加权、Top 选取 | 作为机会排序权重         |
| rate\_actual（实际费率）          | 站→聚合        | **P50 + P25/P75**            | 经营质量主指标      | 易受极端影响，分位更稳      |
| rate\_contract（合同费率）        | 站→聚合        | **P50**（可附分位）                | 合同执行基线       | 合同条款对应口径         |
| fee\_gap = 合同−实际            | 站→聚合        | **P50**                      | 执行偏差→回收空间    | 仅 gap>0 计回收      |
| profit\_per\_order（单利）      | 站→聚合        | **P50 + P25/P75**            | 质量旁证         | 与费率交叉验证          |
| order\_density（订单密度）        | 城/品牌/格子     | **P50 + P25/P75**            | 活跃度、可执行性     | 订单数/当日开单站数       |
| overlap（重叠标签）               | 站→聚合        | 渗透率 %                        | 竞争态势         | 值∈{0,1}；99=未知    |
| brand\_level（品牌档位）          | 站→聚合        | KA/CKA/小散占比                  | 档位结构         | 作为分层与对标维度        |
| ΔP50\_overlap（重叠差）          | 城×品牌        | **ΔP50**                     | 量化压价效应       | 同城×同品牌：无重叠 − 有重叠 |
| ΔP50\_brand（档位差）            | 城内          | **ΔP50**                     | 量化议价梯度       | KA−CKA、CKA−小散    |
| uplift\_benchmark           | 城×品牌/格子     | **ΔP50 × GMV**               | 结构性上拉潜力      | 对标同城目标线          |
| uplift\_contract\_p30/50/70 | 同上          | **fee\_gap P50 × GMV × 回收率** | 立刻可兑现空间      | 回收率 30/50/70 三档  |
| uplift\_conservative        | 同上          | **min(benchmark, p50)**      | 主排序保守值       | 会场可换 p30/p70     |
| N（样本量）                      | 任意聚合        | 计数                           | 可信度          | 结论旁必须展示          |
| coverage（覆盖率）               | 任意聚合        | %                            | 数据质量         | 99/未知单列说明        |
| IQR = P75−P25               | 任意聚合        | 差值                           | 稳定性          | 分布是否集中           |

## 7. 定义注释（统一解释）

* **P50 / P25 / P75**：第 50/25/75 百分位，代表典型水平与典型波动范围；**IQR**=P75−P25。
* **ΔP50**：两组 P50 之差（如无重叠−有重叠、KA−CKA），用“典型值差距”刻画对照差异。
* **fee\_gap**：合同费率 − 实际费率（仅正值视为执行偏差与回收空间）。
* **回收率**：4–8 周内可通过执行/系统治理追回 fee\_gap 的比例；默认三档 **30/50/70%** 同时报告，用于灵敏度场景。
* **uplift\_benchmark**：把低档位/有重叠的 P50 拉到同城目标线（上一级/无重叠）所带来的结构性收益。
* **uplift\_contract**：按回收率折算的执行修正收益；偏“立刻可兑现”。
* **uplift\_conservative**：min(uplift\_benchmark，uplift\_contract\_p50)，用作主排序的保守口径。
* **订单密度**：单位站点订单量（订单数/当日有单站数量），用于判断动作可执行性。
* **覆盖率/99**：标签可用样本占比 / 未知值占比；未知不进主结论，单列报备。

---

## 8. 质量闸门与止损条件

* 覆盖率 < 80% 或 N < 10：不出主结论，降级为观察项并进入数据治理待办。
* 证据层级（若有）：Verified 为主口径，Inferred 仅作灵敏度附注。

---

## 9. 复盘闭环（会后 4–8 周）

* 回看 **回收率实现率**（实际追回 / 预测 uplift\_contract\_p50）。
* 回看 **结构上拉达成率**（实际费率 / 对标目标的贴近度）。
* 依据回看结果，校准下一轮的回收率与显著性阈值，持续调参。

## 10. 群内提数话术（极简三段，可复制）

> ① **先做基本面**：重叠与品牌两条线，全国→Top 省→Top5 城，规模看 Sum（订单/GMV），质量看 **P50±分位**（实际费率/单利/订单密度），并报合同 **P50** 与 **fee\_gap P50**、覆盖率、N。
>
> ② **再找机会**：重叠：同城×同品牌 `ΔP50(费率)=无重叠−有重叠`；品牌：同城相邻档 `ΔP50(费率)`；两条腿都算 `uplift_benchmark` 与 `uplift_contract(30/50/70%)`，**保守取小**，按红绿灯（重叠≥30bp、品牌≥50bp、N≥30）标注。
>
> ③ **不显著再交叉**：品牌腿分 **o=0 / o=1** 单独判；重叠腿必要时合并母品牌后再判。输出**Top 机会清单**（城市×品牌），含 `N/GMV/ΔP50/fee_gap P50/uplift保守/灯色` 与**建议动作**占位。

## 11. 管理层摘要模板（全国 → Top 省区 → Top5 城市）

* **基本面要点**（示例占位）：

  1. 全国 o=1 渗透 X%，对质量指标的影响区间为……；
  2. 品牌档位结构：KA/CKA/小散 占比为……，中位费率分别为……；
  3. 合同执行：fee\_gap 中位为……，Top 省/城为……。
* **机会要点**（示例占位）：

  1. 重叠清单 Top：省A-城1-品牌X，ΔP50=…bp，保守 uplift=……；
  2. 品牌梯度 Top：省B-城2-品牌Y-小散→CKA，ΔP50=…bp，保守 uplift=……；
  3. 会后承诺：TopN 组合、负责人、时间表。
