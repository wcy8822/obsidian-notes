## 3. MCP‑Lite（10条可立即对齐确认的底层原则）

1. 平台定位：**居间/技术服务**，非自营；商品质量与瑕疵担保由商户承担。
2. 试点范围：白名单**低风险SKU**（瓶装水/纸巾/基础车品/洗车券/保养服务），排除冷链/鲜食/药械/危化。
3. 收入口径：商品收入归**商户**；平台按**技术服务费**确认。
4. 确认时点：商品收入与开票=**履约完成**；平台费按**周期**确认。
5. 开票主体：商品票=商户；平台费票=平台；**不得混开**。
6. 优惠券口径：默认**商户折扣**；平台补贴单列营销费用并留痕。
7. 配送责任：**商户自配/有资质第三方**；平台不直接承担配送主体（除另案立项）。
8. 退款逆向：**原路退** + 商品票**红冲** + 分账**负向冲回**，账/票/资**账、票、资金信息保持一致**。
9. 宣传合规：仅用“**预计/目标**××分钟”，暂不启用保证性用语；延时补偿规则公开。
10. 数据最小化：仅采集履约/售后必需信息；三方共享先审后放，留痕≥180天。

## 4. 非油是否/如何“随油品”的**模式决策框架**

### 4.1 最高原则

* **不强制随油品**；默认**撮合**，对满足前置条件的**白名单场站×SKU**可“点状转售”。
* **同一订单保持单一主体**：同一订单仅一种销售主体；必要时系统**自动自动拆分**。

### 4.2 转售前置条件（全部满足方可启用）

* [ ] 经营与资质：平台/上游具备对应类目经营范围与许可
* [ ] 供应链与库存：可追溯批次、WMS台账、有效期/报损SOP
* [ ] 税务与票据：进销项链条清晰、属地化开票可执行、红冲闭环自动化
* [ ] 责任与理赔：召回SOP、保险/先行赔付机制
* [ ] 系统能力：订单级**模式选择器**、**票据引擎**、**三键对账**（订单ID=支付ID=发票ID）

### 4.3 两种模式四流对照（用于会中快速对齐）

| 维度   | 撮合（默认）           | 转售（例外白名单）       |
| ---- | ---------------- | --------------- |
| 销售主体 | 商户               | 平台              |
| 定价权  | 商户主导（可建议）        | 平台主导/联合定价       |
| 开票   | 商品票：商户；平台费票：平台   | 商品票：平台；需进项管理    |
| 收入确认 | 商户：履约完成；平台：服务费按期 | 平台：履约完成；促销归属需定义 |
| 库存   | 商户自持             | 平台入库/寄售台账       |
| 配送责任 | 商户/有资质第三方        | 平台自配或指派第三方      |
| 退款逆向 | 原路退+商户红冲+平台负分账   | 原路退+平台红冲+进销项调整  |
| 合规侧重 | 口径边界/数据最小化       | 资质/质保/WMS/属地开票  |

## D. 四流同图超快对齐（约90秒）

“我们把交易、资金、票据、数据四块画在同一张‘树’上，方便后面讨论不跑题。今天只看主干：关键时点、主体是谁、对账怎么对得上。细节参数我们后面再补。”

## E. 讨论方式（约30秒）

“每个议题我都会给一个**参考做法A**，再给一个**备选做法B**；不求当场定细节，只要看大家**更偏向哪个方向**。不同意见我们都先**记录**，后面用试点数据再复盘。”

## F. 关键议题轻量话术（可边看边说）

1. **平台定位**：“我们更倾向于把平台定位在**居间/技术服务**，先把效率跑起来；如果后面有更稳的SKU，我们**再探索转售**。”
2. **是否随油品**：“油非能不能一把梭一致？我们初步想法是**不强制一致**：油品撮合 → 非油优先撮合；油品转售 → 非油看**条件成熟度**，成熟就跟上；不成熟就先走撮合。”
3. **订单关系**：“为了降低开票和对账复杂度，**一个订单一个主体**，需要的话系统帮用户**自动拆单**，减少错配。”
4. **发票与时点**：“更偏向于在**履约完成**再开票、再确认收入；这样和售后、退款能**一条线**走下来。”
5. **优惠与补贴**：“先按**商户折扣**来处理最简单；如果平台侧有补贴，我们**单列**出来，后面统一复盘效果。”
6. **配送与责任**：“先以**商户自配/有资质第三方**为主；如果有更合适的方案，后面我们开专项再细化。”

## G. 伙伴可能关心的问题 & 温和回应

* **为什么不直接全跟油品？**
  “油和非油在票据、库存、责任上**差异挺大**。我们先把**共性**的部分靠拢，不同的部分**分层推进**，避免一开始就把复杂度做高。”
* **混合模式会不会搞乱开票？**
  “我们准备在系统里做**模式选择**和**自动自动拆分**，再配一个简单的**票据映射**，尽量把风险挡在系统层。”
* **税金和现金流会不会有压力？**
  “试点阶段我们**小范围跑**，每周看一次‘进销项差、在途天数、红冲率’，不冒进，有结论再扩大。”
* **消费者权益怎么兜？**
  “非承诺式表达、**预计时效**+**补偿规则**，质量责任由**商品主体**承担；平台这边把**先行处理**机制准备好。”
